The Big Idea
What is a linked list and why do we need it?

Think of a treasure hunt. Each clue does not tell you where the treasure is directly — it tells you where the next clue is. A linked list works the same way. Each item says: "here is my data, and here is where the next item lives."

You already know
Array
Items sit next to each other. Like seats in a row. To add a seat in the middle, everyone shifts.
→
The problem
Shifting is slow
Insert 15 between 10 and 20 — you must move 20, 30, 40… That is a lot of work.
→
The solution
Linked List
Items live anywhere in memory. To insert, just change two "directions". No shifting needed.

🛑 The two most important rules — memorise these:
A pointer value of –1 means "there is no next item" (end of list).
head index = –1 means the list is completely empty.
first free index = –1 means the list is completely full (no space left).

02
Constants
Values that never change

A constant is just a name we give to a value so we do not have to write the number everywhere. Instead of writing –1 in every function, we write EMPTY. Much easier to read.

# This is the only constant we need.
# –1 means "nothing here" or "no next item".
EMPTY = -1
LineWhat it means in plain English
EMPTY = -1We create a name EMPTY that means "no value / does not exist." We use this instead of writing -1 every time.
💡 Why not just write –1 everywhere?
If you use the name EMPTY, your code is easier to read. "pointer = EMPTY" is clearer than "pointer = -1". The examiner also sees you understand what –1 means.
03
The Node Class
The blueprint for one item in the list

A class is like a recipe card. It describes what one node looks like. Every node has exactly two things: its value (the actual data) and a next index (which slot comes after it).

class Node:
    def __init__(self, value="", next_index=EMPTY):
        self.value      = value
        self.next_index = next_index
LineWhat it means in plain English
class Node:We are creating a blueprint called Node. Every slot in the list will use this blueprint.
def __init__(...)This runs automatically when we create a new Node. It sets up its starting values.
value=""The actual data stored. Starts as empty text by default.
next_index=EMPTYThe slot number of the next item. Starts as –1 (meaning: no next item yet).
self.value = valueSave the value into this node.
self.next_index = next_indexSave the next slot number into this node.
🧠 Think of it like a box with two compartments:
Left side = the actual data (a number, a name…)
Right side = the slot number of the next box in the chain
04
Create and Initialise the List
Setting everything up before we use it

Before we can use the linked list, we must set it up. We create an array of empty nodes and link them all together in a "free chain" — like a waiting room where empty seats know which seat is next.

# How many slots our list can hold
MAX_SIZE = 7

# The two global variables we always need
head_index      = EMPTY   # –1 means the list is empty right now
first_free_index = 0      # first empty slot is at position 0

# Create the array: 7 empty nodes
slots = [Node() for _ in range(MAX_SIZE)]

# Link all the free slots together
def setup_list():
    global head_index, first_free_index, slots
    head_index       = EMPTY
    first_free_index = 0
    for i in range(MAX_SIZE - 1):
        slots[i].next_index = i + 1
    slots[MAX_SIZE - 1].next_index = EMPTY
LineWhat it means in plain English
head_index = EMPTYNo items in the list yet. The list is empty. –1 signals this.
first_free_index = 0The first empty slot we can use is at position 0.
slots = [Node() ...]Create 7 empty boxes (nodes), all blank to start.
for i in range(MAX_SIZE - 1):Go through every slot except the last one.
slots[i].next_index = i + 1Each free slot points to the one after it. Slot 0 → 1, slot 1 → 2, etc.
slots[MAX_SIZE-1].next_index = EMPTYThe last slot has no "next". So it gets –1 to say "I am the end."

After setup, the array looks like this — all 7 slots are free and linked:

Slot.value.next indexNote
0""1← first_free_index starts here
1""2free slot, points to next free
2""3free slot

📋 What does setup_list() do?

It prepares the linked list so it is ready to use. It says: "the list is empty" (head_index = –1) and links all 7 empty slots together so we know which ones are free. Think of it as setting up an empty classroom with chairs all connected by a rope — you can grab the first chair and follow the rope to find the next free one.

05
Operations
Insert · Traverse · Find · Delete

① Insert at the Front

This puts a new item at the beginning of the list. The new item becomes the new head. Do this in 4 steps:

1
Check if full. If first_free_index == –1, there are no empty slots. Print a message and stop.
2
Grab a free slot. Save first_free_index as your new slot. Then move first_free_index forward to the next free slot.
3
Store the data. Put the value in that slot. Make the new slot point to the old head (the old first item).
4
Update the head. The new slot is now the first item. Set head_index to the new slot.
def insert_at_front(value):
    global slots, head_index, first_free_index

    if first_free_index == EMPTY:          # Step 1: is the list full?
        print("List is full — cannot insert")
        return

    new_slot         = first_free_index                   # Step 2a: grab the slot
    first_free_index = slots[first_free_index].next_index # Step 2b: move free pointer forward

    slots[new_slot].value      = value       # Step 3a: store the data
    slots[new_slot].next_index = head_index  # Step 3b: point to old head
    head_index = new_slot                   # Step 4: new slot is the new head
LineWhat it means in plain English
new_slot = first_free_indexRemember which slot we are about to use.
first_free_index = slots[...].next_indexMove the "next free" pointer to the slot after the one we just took.
slots[new_slot].value = valuePut the actual number/word into our new slot.
slots[new_slot].next_index = head_indexThe new item points to whatever was the old first item.
head_index = new_slotNow this new slot is the head (the first item in the list).
📋 What does insert_at_front() do?

It adds one item to the beginning of the list. Imagine a queue — instead of going to the back, this person cuts to the front. The old "first person" becomes second. The new person is now first.

② Traverse — Read the Whole List

To read every item, start at head_index and keep following next_index until you reach –1 (the stop sign).

def print_list():
    global slots, head_index

    current = head_index              # start at the first item
    while current != EMPTY:           # keep going until we hit –1
        print(slots[current].value)   # print this item's value
        current = slots[current].next_index  # follow the arrow to next item
LineWhat it means in plain English
current = head_indexStart at the first item in the list.
while current != EMPTYKeep looping as long as we have not reached the end (–1).
print(slots[current].value)Print the value stored in the current slot.
current = slots[current].next_indexMove to the next slot by following the arrow.
📋 What does print_list() do?

It reads and prints every item in the list, from first to last. It follows the chain of arrows until it sees –1 and then stops. Like following a treasure hunt until there are no more clues.

③ Find — Search for a Value

def find_item(target):
    global slots, head_index

    current = head_index
    while current != EMPTY:
        if slots[current].value == target:
            print(f"Found {target} at slot {current}")
            return current
        current = slots[current].next_index

    print(f"{target} is not in the list")
    return EMPTY
LineWhat it means in plain English
if slots[current].value == targetIs this the item we are looking for?
return currentYes! Return the slot number where it lives.
print(... "not in the list")We went through every item and did not find it.
return EMPTYReturn –1 to signal "not found."
📋 What does find_item() do?

It searches the list for a specific value. It checks each item one by one. If it finds it, it says where. If not, it says "not found." Like looking for a name in a phone book — one line at a time.

④ Delete — Remove an Item and Recycle the Slot

We use two "walkers": current (the item we are checking) and previous (the item just before it). When we find the target, we make previous skip over current. Then we recycle the slot back to the free list.

def delete_item(target):
    global slots, head_index, first_free_index

    current  = head_index
    previous = EMPTY          # no previous item at the start

    # Walk until we find the target
    while current != EMPTY and slots[current].value != target:
        previous = current
        current  = slots[current].next_index

    if current == EMPTY:
        print(f"{target} not found")
        return

    # Re-link: skip over the deleted item
    if previous == EMPTY:                              # deleting the very first item
        head_index = slots[current].next_index
    else:                                              # deleting a middle or last item
        slots[previous].next_index = slots[current].next_index

    # Recycle: return this slot to the free list
    slots[current].value      = ""
    slots[current].next_index = first_free_index
    first_free_index          = current
LineWhat it means in plain English
previous = EMPTYWe start with no "previous item" because we are at the very beginning.
previous = current; current = ...Move both walkers forward together, one step at a time.
if previous == EMPTYWe are deleting the very first item. Update the head to the next item.
slots[previous].next_index = slots[current].next_indexThe item before the deleted one now points to the item after it. Skip over the deleted one. Like removing link B from chain A→B→C so it becomes A→C.
slots[current].value = ""Clear the data from the deleted slot.
slots[current].next_index = first_free_indexThe recycled slot points to the old first free slot.
first_free_index = currentThe recycled slot is now the first free slot (available for future inserts).
📋 What does delete_item() do?

It finds a value in the list and removes it. Then it puts the empty slot back into the "free pool" so it can be reused later. Think of it like removing someone from a queue — the people on either side of them step together, and the removed person's spot is marked as available again.

💡 The skip-over trick visualised:
Before: A → B → C (deleting B)
After: A → C (A now points directly to C, B is gone)
Code does this: slots[previous].next_index = slots[current].next_index

2D Array Linked Lists
What the exam actually uses — explained simply

The Cambridge exam does NOT use a Node class. Instead, it uses a 2D array (a grid). Each row is one slot. Column 0 = the data. Column 1 = the next slot number. That is it.

🧠 Translation table — same idea, different syntax:

slots[i].value → LinkedList[i][0]
slots[i].next_index → LinkedList[i][1]
head_index → FirstNode
first_free_index → FirstEmpty

The exam code uses LinkedList[i][0] for data and LinkedList[i][1] for the next pointer. The logic is exactly the same as the Node class — just different brackets.

# 2D array version of "insert at front" — exam style
def InsertData(value):
    global LinkedList, FirstNode, FirstEmpty

    if FirstEmpty == -1:           # list is full
        print("List is full")
        return

    new_slot               = FirstEmpty
    FirstEmpty             = LinkedList[FirstEmpty][1]  # move free pointer
    LinkedList[new_slot][0] = value                   # store data
    LinkedList[new_slot][1] = FirstNode                # point to old head
    FirstNode              = new_slot                  # update head
LineWhat it means in plain English
LinkedList[FirstEmpty][1]Read the next-slot value of the current free slot. This tells us which slot is free after this one.
LinkedList[new_slot][0] = valueStore the actual data in column 0 of this slot.
LinkedList[new_slot][1] = FirstNodePoint to the old head (column 1 = the next-slot number).

Ex 1Write a procedure to insert one item at the front
Easy[5 marks]▼ show
A linked list is stored in a 2D array called LinkedList with 10 slots. Column 0 stores the data, column 1 stores the pointer to the next slot. FirstNode holds the index of the first data item. FirstEmpty holds the index of the first free slot. Both are initially set to the correct starting values.

The procedure InsertFront(value) takes one integer as a parameter and inserts it at the front of the linked list. If the list is full, it should output a suitable message and stop.
  • Write program code for InsertFront(value). [5]
Step 1: Check if FirstEmpty == –1 (full). Step 2: Save FirstEmpty as your new slot. Step 3: Move FirstEmpty forward. Step 4: Store the value in column 0. Step 5: Point the new slot to the old FirstNode (column 1). Step 6: Update FirstNode to the new slot.
Model Answer
def InsertFront(value):
    global LinkedList, FirstNode, FirstEmpty

    if FirstEmpty == -1:
        print("List is full")
        return

    new_slot                = FirstEmpty
    FirstEmpty              = LinkedList[FirstEmpty][1]
    LinkedList[new_slot][0] = value
    LinkedList[new_slot][1] = FirstNode
    FirstNode               = new_slot

Mark scheme [5]:
[1] check FirstEmpty == –1, output message and stop
[1] save FirstEmpty as new_slot
[1] advance FirstEmpty to LinkedList[FirstEmpty][1]
[1] store value in column 0; set column 1 to old FirstNode
[1] update FirstNode to new_slot
Ex 2Write a function to output the list and count items
Easy[4 marks]▼ show
Using the same linked list as Exercise 1.

(i) Write a procedure called OutputList() that outputs every data item in the list in order, starting from FirstNode.

(ii) Write a function called CountItems() that returns the total number of data items currently in the list as an integer.
  • Write program code for OutputList(). [2]
  • Write program code for CountItems(). [2]
Both functions start at FirstNode and follow column 1 until –1. OutputList prints each value. CountItems uses a counter that adds 1 each time the loop runs, then returns it at the end.
Model Answer
# (i)
def OutputList():
    global LinkedList, FirstNode

    current = FirstNode
    while current != -1:
        print(LinkedList[current][0])
        current = LinkedList[current][1]

# (ii)
def CountItems():
    global LinkedList, FirstNode

    count   = 0
    current = FirstNode
    while current != -1:
        count  += 1
        current = LinkedList[current][1]
    return count

Mark scheme:
[1] OutputList: correct loop following column 1 until –1
[1] OutputList: prints column 0 of each node
[1] CountItems: counter starts at 0, increments each iteration
[1] CountItems: returns count after loop ends
Ex 3Write a procedure to search for a value and report if found
Intermediate[4 marks]▼ show
Using the same linked list. A procedure called SearchList(target) takes one integer as a parameter.

It searches the linked list for target. If found, it outputs the message "Found: " followed by the slot number where it is stored. If not found, it outputs "Not found".
  • Write program code for SearchList(target). [4]
Start at FirstNode. Walk the list. Each step: check if column 0 equals target. If yes → print the slot number (that is the variable current) and stop. If the loop ends without finding it → print "Not found".
Model Answer
def SearchList(target):
    global LinkedList, FirstNode

    current = FirstNode

    while current != -1:
        if LinkedList[current][0] == target:
            print("Found: slot", current)
            return
        current = LinkedList[current][1]

    print("Not found")

Ex 4Write a procedure to delete a node and recycle the slot
Intermediate[6 marks]▼ show
Using the same linked list. The procedure DeleteItem(target) takes one integer as a parameter.

It searches for the first node whose data equals target. When found, it removes that node from the list by updating the pointers of the surrounding nodes. It then returns the deleted slot to the free list by updating FirstEmpty. You can assume target is always present in the list.
  • Write program code for DeleteItem(target). [6]
You need two walkers: current (starts at FirstNode) and previous (starts at –1). Walk until you find target. Then: if previous == –1, update FirstNode. Otherwise, set previous's column 1 to current's column 1 (skip over current). Finally: clear the slot's data, set its pointer to old FirstEmpty, set FirstEmpty = current.
Model Answer
def DeleteItem(target):
    global LinkedList, FirstNode, FirstEmpty

    current  = FirstNode
    previous = -1

    while current != -1 and LinkedList[current][0] != target:
        previous = current
        current  = LinkedList[current][1]

    if previous == -1:
        FirstNode = LinkedList[current][1]
    else:
        LinkedList[previous][1] = LinkedList[current][1]

    LinkedList[current][0] = -1
    LinkedList[current][1] = FirstEmpty
    FirstEmpty = current

Mark scheme [6]:
[1] two pointers declared: current = FirstNode, previous = –1
[1] loop condition: current != –1 AND column 0 != target
[1] previous and current both move forward inside the loop
[1] if previous == –1: FirstNode updated to current's next
[1] else: LinkedList[previous][1] = LinkedList[current][1]
[1] slot recycled: data cleared, pointer = old FirstEmpty, FirstEmpty = current
Ex 5Write a procedure to insert multiple items and then remove one
Hard[8 marks]▼ show
A linked list is stored in a 2D array called LinkedList with 8 slots. FirstNode and FirstEmpty are already initialised correctly.

(i) Write a procedure LoadData() that asks the user to input 3 positive integers one at a time and inserts each at the front of the linked list. If the list becomes full before all 3 are inserted, output "List full" and stop.

(ii) Write a function GetLast() that traverses the list and returns the data value of the last node (the node whose pointer is –1). If the list is empty, return –1.

(iii) Write program code for the main program that calls LoadData(), then calls GetLast() and outputs the result with the label "Last item:".
  • Write program code for LoadData(). [4]
  • Write program code for GetLast(). [3]
  • Write main program code. [1]
LoadData: loop 3 times, same pattern as InsertFront but check full first. GetLast: if FirstNode == –1, return –1. Otherwise walk the list until you find a node whose column 1 is –1 — that is the last node. Return its column 0 (the data).
Model Answer
# (i)
def LoadData():
    global LinkedList, FirstNode, FirstEmpty

    for _ in range(3):
        if FirstEmpty == -1:
            print("List full")
            return
        value                   = int(input("Enter a number: "))
        new_slot                = FirstEmpty
        FirstEmpty              = LinkedList[FirstEmpty][1]
        LinkedList[new_slot][0] = value
        LinkedList[new_slot][1] = FirstNode
        FirstNode               = new_slot

# (ii)
def GetLast():
    global LinkedList, FirstNode

    if FirstNode == -1:
        return -1

    current = FirstNode
    while LinkedList[current][1] != -1:
        current = LinkedList[current][1]
    return LinkedList[current][0]

# (iii) Main program
LoadData()
print("Last item:", GetLast())

Mark scheme:
LoadData [4]:
  [1] loop runs exactly 3 times
  [1] checks FirstEmpty == –1, outputs message and stops
  [1] takes input; new_slot = FirstEmpty; FirstEmpty advanced
  [1] stores data; sets column 1 to old FirstNode; updates FirstNode
GetLast [3]:
  [1] handles empty list (FirstNode == –1), returns –1
  [1] loop walks until LinkedList[current][1] == –1
  [1] returns LinkedList[current][0] (the last data value)
Main [1]:
  [1] calls LoadData() then prints "Last item:" with GetLast() result
