Printing variables — f-strings explained
How to insert variables into text, format decimals, and what every symbol means.
What is an f-string? An f-string is a string that can hold variables inside it. You put the letter f before the opening quote, and then put variable names inside curly braces { }. Python replaces the curly braces with the actual value of the variable when the program runs.
Anatomy of an f-string — what every part does:
f"Hello, my name is {name} and I am {age} years old."
f → This single letter before the quote tells Python: "this string contains variables". Without the f, Python treats everything as plain text and the curly braces print literally.
" " → The quotes wrap the whole string. You can use single quotes ' or double quotes " — both work. Everything inside the quotes is your text, except what's inside { }.
{ } → Curly braces are placeholders. Python looks up the variable name inside the braces and replaces the whole {variable} with its value. You can put any variable here — string, integer, float, boolean.
name → The variable name. No quotes around it — it's not a string, it's a reference to a variable that already exists. Python goes and finds what's stored in that variable.
Formatting decimal numbers — the :.2f syntax:
f"Area: {area:.2f}"
area → The variable whose value you want to display.
: → The colon separates the variable name from the formatting instructions. Think of it as saying "now tell me HOW to display it".
. → The dot means "I want to control the decimal places".
2 → The number of decimal places you want. Change this to 0, 1, 3 — whatever you need. Cambridge exams usually ask for 2 decimal places.
f → Stands for "float". It tells Python: this is a decimal number, display it with the number of decimal places I specified. Always write f at the end.
The full pattern to memorise {variable:.Xf} where X is how many decimal places you want.
{area:.2f} → 2 decimal places
{area:.0f} → 0 decimal places (rounds to whole number)
{area:.4f} → 4 decimal places
All forms — every way to print a variable:
name  = "Ali"
age   = 17
score = 3.14159
done  = True

# 1. Plain variable — no formatting
print(f"Name: {name}")          # Name: Ali
print(f"Age: {age}")           # Age: 17

# 2. Decimal formatting
print(f"Score: {score}")        # Score: 3.14159  (no control)
print(f"Score: {score:.2f}")   # Score: 3.14     (2 d.p.)
print(f"Score: {score:.0f}")   # Score: 3        (0 d.p. — rounds)
print(f"Score: {score:.4f}")   # Score: 3.1416   (4 d.p.)

# 3. Multiple variables in one line
print(f"Hello {name}, you are {age} years old")

# 4. Expression inside { } — Python calculates it
print(f"In 5 years: {age + 5}")  # In 5 years: 22
print(f"Double: {age * 2}")      # Double: 34

# 5. Without f-string (old way — harder to read)
print("Name: " + name)           # works but messy
print("Age: "  + str(age))        # must convert int to str first
Common mistake — forgetting the f If you write print("Hello {name}") without the f, Python prints literally: Hello {name}. The curly braces do nothing without the f in front of the quote. Always check that f is there.
Cambridge exam rule Whenever the question says "output to 2 decimal places" — use :.2f inside your f-string. This comes up in almost every calculation question: areas, averages, percentages.
Substring & string slicing — all cases
Cambridge uses SUBSTRING(s, start, length). Python uses slicing: s[start:stop]. They work slightly differently — know both.
Critical difference between Cambridge and Python Cambridge SUBSTRING(s, 0, 3) means: start at index 0, take 3 characters.
Python s[0:3] means: start at index 0, stop BEFORE index 3.
Both give the same result — but the second number means different things. Cambridge = how many characters. Python = where to stop. 
# Our example string for all cases below:
s = "Cambridge"
#   C a m b r i d g e
#   0 1 2 3 4 5 6 7 8   ← index numbers
What you want Cambridge pseudocode Python slicing Result Explanation
First 3 characters SUBSTRING(s, 0, 3) s[0:3] "Cam" Start at 0, stop before index 3
First character only SUBSTRING(s, 0, 1) s[0:1] or s[0] "C" Index 0 only. s[0] is shorthand for one character
Last character only SUBSTRING(s, 8, 1) s[-1] "e" -1 always means the last character — counts from the end
Last 3 characters SUBSTRING(s, 6, 3) s[-3:] "dge" Start 3 from the end, take everything to the finish
Middle section SUBSTRING(s, 2, 4) s[2:6] "mbri" Start at index 2, stop before index 6 (= 4 chars)
Everything from index 3 SUBSTRING(s, 3, LEN(s)-3) s[3:] "bridge" No stop number = go all the way to the end
Everything up to index 5 SUBSTRING(s, 0, 5) s[:5] "Cambr" No start number = begin from index 0 automatically
Whole string s s[:] "Cambridge" No start, no stop = copy entire string
Length of string LEN(s) len(s) 9 Counts all characters including spaces
The formula to convert Cambridge → Python slicing Cambridge: SUBSTRING(s, start, length)
Python: s[start : start + length]

Example: SUBSTRING(s, 2, 4) → s[2 : 2+4] → s[2:6] → "mbri"
Just add the start and the length together to get the stop number.

P1 Temperature Converter with Validation
Intermediate [7 marks] ▼ expand
A program reads a temperature in Celsius from the user and converts it. If the temperature is below absolute zero (–273), the program must reject the input and ask again.
DECLARE celsius     : REAL
DECLARE fahrenheit  : REAL
DECLARE kelvin      : REAL

celsius ← USERINPUT
WHILE celsius < -273 DO
    OUTPUT "Invalid. Enter again:"
    celsius ← USERINPUT
ENDWHILE

fahrenheit ← (celsius * 9 / 5) + 32
kelvin     ← celsius + 273.15

OUTPUT "Fahrenheit: " & fahrenheit
OUTPUT "Kelvin: "     & kelvin
  • No DECLARE lines in Python [1]
  • Input with float() conversion [1]
  • WHILE loop with correct condition [2]
  • Both formulas translated correctly [2]
  • Both outputs using f-strings with :.2f [1]
HintThe WHILE loop needs the input both before the loop (to check the first time) and inside the loop (to ask again). The condition celsius < -273 stays the same — just translate WHILE ... DO to while ...: and remove ENDWHILE.
celsius = float(input("Enter temperature in Celsius: ")) while celsius < -273: print("Invalid. Enter again:") celsius = float(input("Enter temperature in Celsius: ")) fahrenheit = (celsius * 9 / 5) + 32 kelvin = celsius + 273.15 print(f"Fahrenheit: {fahrenheit:.2f}") print(f"Kelvin: {kelvin:.2f}")

P4 String Analysis Function
Hard [10 marks] ▼ expand
A function analyses a string and counts vowels, consonants and spaces separately.
FUNCTION AnalyseString(text : STRING)
    DECLARE vowels, consonants, spaces : INTEGER
    DECLARE i : INTEGER
    DECLARE ch : STRING
    vowels     ← 0
    consonants ← 0
    spaces     ← 0

    FOR i ← 0 TO LEN(text) - 1
        ch ← LCASE(SUBSTRING(text, i, 1))
        IF ch = " " THEN
            spaces ← spaces + 1
        ELSE IF ch = "a" OR ch = "e" OR ch = "i"
                OR ch = "o" OR ch = "u" THEN
            vowels ← vowels + 1
        ELSE
            consonants ← consonants + 1
        ENDIF
    NEXT i

    OUTPUT "Vowels: "     & vowels
    OUTPUT "Consonants: " & consonants
    OUTPUT "Spaces: "     & spaces
ENDFUNCTION

DECLARE sentence : STRING
OUTPUT "Enter a sentence:"
sentence ← USERINPUT
CALL AnalyseString(sentence)
  • Correct def syntax, no type annotations [1]
  • Initialise three counters to 0 [1]
  • range(len(text)) for the loop [1]
  • SUBSTRING(text, i, 1) → text[i] or text[i:i+1] [1]
  • LCASE → .lower() [1]
  • Correct if/elif/else with == and or [2]
  • Three counters updated correctly [1]
  • Three print statements [1]
  • Correct function call without CALL [1]
HintFOR i ← 0 TO LEN(text) - 1 → range(len(text)). SUBSTRING(text, i, 1) gets one character — in Python that's text[i]. LCASE → .lower(). The ELSE IF with multiple OR conditions → elif ch == "a" or ch == "e" or ch == "i" or ch == "o" or ch == "u".
def AnalyseString(text): vowels = 0 consonants = 0 spaces = 0 for i in range(len(text)): ch = text[i].lower() if ch == " ": spaces += 1 elif ch in "aeiou": # cleaner than 5 OR conditions vowels += 1 else: consonants += 1 print(f"Vowels: {vowels}") print(f"Consonants: {consonants}") print(f"Spaces: {spaces}") sentence = input("Enter a sentence: ") AnalyseString(sentence)

P6 Recursive Factorial + Function Calls
Hard [10 marks] ▼ expand
A program contains a recursive factorial function and a procedure that prints a table of factorials from 1 to n.
FUNCTION Factorial(n : INTEGER) : INTEGER
    IF n <= 1 THEN
        RETURN 1
    ELSE
        RETURN n * Factorial(n - 1)
    ENDIF
ENDFUNCTION

PROCEDURE PrintTable(limit : INTEGER)
    DECLARE i : INTEGER
    FOR i ← 1 TO limit
        OUTPUT i & "! = " & Factorial(i)
    NEXT i
ENDPROCEDURE

DECLARE n : INTEGER
OUTPUT "How many factorials to show?"
n ← USERINPUT
CALL PrintTable(n)
  • Recursive function: correct def, base case, recursive case [3]
  • RETURN → return (lowercase) [1]
  • Procedure: correct def, no return statement [2]
  • range(1, limit+1) — includes limit [1]
  • Factorial(i) called inside the loop [1]
  • Main: int() input, PrintTable(n) without CALL [2]
HintThe recursive function calls itself: Factorial(n-1). The base case stops it: if n <= 1 return 1. The procedure has no return — it just prints. FOR i ← 1 TO limit → range(1, limit+1). No CALL keyword when calling PrintTable.
def Factorial(n): if n <= 1: return 1 else: return n * Factorial(n - 1) def PrintTable(limit): for i in range(1, limit + 1): print(f"{i}! = {Factorial(i)}") n = int(input("How many factorials to show? ")) PrintTable(n)
P7 Global Variable — Score Counter
Hard [9 marks] ▼ expand
A quiz game uses a global score counter that is updated by a procedure each time the player answers correctly.
GLOBAL score ← 0
GLOBAL highScore ← 0

PROCEDURE AddPoints(points : INTEGER)
    score ← score + points
    IF score > highScore THEN
        highScore ← score
    ENDIF
ENDPROCEDURE

PROCEDURE ResetScore()
    score ← 0
ENDPROCEDURE

CALL AddPoints(10)
CALL AddPoints(25)
OUTPUT "Score: " & score
OUTPUT "High score: " & highScore
CALL ResetScore()
OUTPUT "After reset: " & score
  • Both global variables declared at module level [1]
  • AddPoints: global score, global highScore declared inside [2]
  • score updated correctly [1]
  • highScore updated with correct condition [1]
  • ResetScore: global score declared, score set to 0 [2]
  • All calls without CALL keyword [1]
  • All three print statements correct [1]
HintAny function that CHANGES a global variable needs "global varname" at the top. AddPoints changes both score and highScore — so it needs global score, global highScore. ResetScore changes score — so it needs global score. Functions that only READ a global don't need the global keyword.
score = 0 highScore = 0 def AddPoints(points): global score, highScore score += points if score > highScore: highScore = score def ResetScore(): global score score = 0 AddPoints(10) AddPoints(25) print(f"Score: {score}") print(f"High score: {highScore}") ResetScore() print(f"After reset: {score}")
