01
What Even Is a Hash Table? — Start From Zero
One analogy. One formula. That's all you need.
📬 The simplest analogy: a post office with numbered boxes Imagine a post office with 10 mailboxes, numbered 0 to 9.

Every person has an ID number. When they get mail, the post worker doesn't check every box. Instead, they do one calculation:

box_number = person_ID % 10

Person with ID 234 → 234 % 10 = 4 → mail always goes to Box 4.
Person with ID 7 → 7 % 10 = 7 → mail always goes to Box 7.

To find mail? Same calculation. One step. No searching. That's a hash table.
THE POST OFFICE — 10 boxes, numbered 0 to 9
📖 The formal definition A hash table stores data as key-value pairs. A hash function converts the key into an array index. This gives extremely fast insertion, lookup, and deletion — often in constant time O(1).
The 5 vocabulary words Cambridge will test — know every one 🧊
TermPlain English definitionExample
Key The unique identifier — what you search by. Usually an integer. Student ID: 4521
Value The actual data you store alongside the key. Student name: "Sami"
Hash Function A formula: input = key, output = array index. Cambridge always uses key % table_size. 4521 % 10 = 1 → store at index [1]
Hash Address The index the hash function produces. This is WHERE in the array the data lives. Index [1] is the hash address for key 4521
Collision When TWO different keys produce the SAME hash address. Both want slot [x]. Problem. 4521 % 10 = 1 AND 6231 % 10 = 1 → COLLISION at [1]
02
The Hash Function — One Formula, Understand It Once
hash_address = key % table_size
🧮 The formula — memorise it
hash_address = key % table_size ← this one line is the entire hash function
The % symbol means: divide the key by the table size, throw away the whole number, and keep only the remainder. The result is always between 0 and (table_size − 1), so it always fits inside the array.

Example: 234 % 10 → 234 ÷ 10 = 23 remainder 4 → address is 4. That's it. That's the whole formula.

03
3 Ways to Handle a Collision 💥
Chaining · Overflow Area · Neighbouring Slots
Method 1
Chaining — Build a linked list at the collision spot 🔗

Simple idea: When two keys land on the same slot, don't panic. Just make a linked list at that slot. Every new collision victim gets added to the end of the list. The slot stores a pointer to the head of that list.

Keys 14, 24, 34 all hash to slot [4] → they form a chain
How it works — 3 rules
Normal insert
→
Hash key → slot is empty → store item there directly.
Collision insert
→
Slot taken → create new linked list node → append to the chain at that slot.
Search
→
Hash key → jump to slot → walk the list node by node → key matches? Return value. Reach null? Not found.
🐍 Method 1 Python — Chaining with a list at each slot
The simplest possible implementation: each slot is a Python list that holds all key-value pairs that hash there.
SIZE = 10
HashTable = [[] for _ in range(SIZE)]   # 10 empty lists — one per slot

def Hash(key):
    return key % SIZE

def Insert(key, value):
    global HashTable
    address = Hash(key)
    HashTable[address].append((key, value))   # add (key,value) tuple to this slot's list

def Search(key):
    global HashTable
    address = Hash(key)
    for (k, v) in HashTable[address]:          # walk the list at that slot
        if k == key:
            return v                             # ✅ found!
    return "Not found"                        # reached end of list, not there

# Example:
Insert(14, "Ali")    # goes to slot [4]
Insert(24, "Nour")   # COLLISION → also slot [4], appended to list
Insert(34, "Ziad")   # COLLISION → also slot [4], appended again
print(Search(34))    # → "Ziad"
💡 Wait — are we using a linked list or a Python list here? Good question. In theory, chaining uses a linked list (nodes connected by pointers). But in Python, we use a regular list (like an array) instead — and that's totally fine. Why? Because Python lists are dynamic: they grow and shrink automatically. You don't need to manage pointers or nodes manually. .append() adds a new item to the end in one line — Python handles all the memory behind the scenes. So HashTable[address].append((key, value)) achieves exactly what a linked-list node-insertion would do, just with much simpler code. The concept is chaining; the tool is Python's built-in dynamic list.
Method 2
Overflow Area — A VIP waiting zone for collisions 🗃️

The array is split into two zones: the main area (normal slots) and the overflow area (reserved only for collision victims). When a collision happens, the victim goes to the overflow area, and the main slot stores a pointer to find it there.

Table split: Main area [0–6] · Overflow area [7–9] · Keys 14, 24, 34 all hash to [4]
How it works — 3 rules
Array split
→
Divide the array: slots 0 to N = main area. Last few slots = overflow area (reserved for collisions ONLY).
Collision insert
→
Main slot taken → put collision victim in next free overflow slot. The main slot stores a pointer to it.
Search
→
Hash key → check main slot. Match? Return value. Has overflow pointer? Follow it. Chain ends at null = "Not found".
🐍 Method 2 Python — Overflow Area
We split the array: indices 0 to MAIN_SIZE-1 are main, MAIN_SIZE onwards are overflow. Each slot stores (key, value, overflow_index).
MAIN_SIZE = 7     # main area: slots 0-6
OVF_SIZE  = 3     # overflow: slots 7-9
TOTAL     = MAIN_SIZE + OVF_SIZE   # = 10

# Each slot is a list of 3 values: [key, value, overflow_pointer]
# key = -1 means this slot is empty.
# overflow_pointer = -1 means "no overflow from this slot".
# overflow_pointer = 8 means "there's a collision victim stored at index 8".
HashTable = [[-1, "", -1] for _ in range(TOTAL)]

# Example of what a filled slot looks like:
#   [14, "Ali", 8]  → key=14, value="Ali", collision victim is at index 8
#   [24, "Nour", -1] → key=24, value="Nour", no further overflow
#   [-1, "", -1]    → empty slot

ovf_next = MAIN_SIZE   # next free overflow slot (starts at 7)

def Hash(key):
    return key % MAIN_SIZE

def Insert(key, value):
    global HashTable, ovf_next
    address = Hash(key)

    if HashTable[address][0] == -1:      # main slot is empty → store here
        HashTable[address] = [key, value, -1]
    else:                                 # collision! walk to end of overflow chain
        current = address
        while HashTable[current][2] != -1: # [2] is the overflow pointer
            current = HashTable[current][2]
        HashTable[current][2] = ovf_next  # point last node → new overflow slot
        HashTable[ovf_next] = [key, value, -1]
        ovf_next += 1

def Search(key):
    global HashTable
    current = Hash(key)
    while current != -1:
        if HashTable[current][0] == key:
            return HashTable[current][1]  # ✅ found!
        current = HashTable[current][2]   # follow overflow pointer
    return "Not found"
Method 3
Neighbouring Slots — "No space here? Try next door!" 🔎

The simplest structure of all — just one flat array. When a collision happens, don't build lists or use a separate zone. Just check the next slot. If that's also taken, try the one after. Keep going until you find an empty slot. Also called linear probing.

Inserting keys 14, 24, 34 — all hash to slot [4] — into a size-10 table
How it works — 4 rules
Normal insert
→
Hash key → slot is empty → insert there. Done.
Collision insert
→
Slot taken → check [address+1]. Still taken? Check [address+2]. Keep going until you find an empty slot.
Wrap-around
→
If you hit slot [9] and it's full, wrap back to [0] using: next = (current + 1) % size.
Search stop rule
→
If you hit an empty slot while probing → STOP. Return "Not found". Empty = the key was never stored beyond here.
🐍 Method 3 Python — Linear Probing (Neighbouring Slots)
The cleanest version. One flat array, -1 means empty. Probe forward with wrap-around using % SIZE.
SIZE = 10
HashTable = [-1] * SIZE   # -1 means "empty slot"

def Hash(key):
    return key % SIZE

def Insert(key):
    global HashTable
    address = Hash(key)
    while HashTable[address] != -1:          # slot is occupied — probe next
        address = (address + 1) % SIZE       # the wrap-around formula
    HashTable[address] = key                  # found an empty slot — insert here

def Search(key):
    global HashTable
    address = Hash(key)
    start = address                           # remember where we started (avoid infinite loop)
    while HashTable[address] != -1:          # empty slot? STOP — key not here
        if HashTable[address] == key:
            return address                   # ✅ found it!
        address = (address + 1) % SIZE
        if address == start:                  # full circle, table is full
            break
    return -1                               # not found

# Example trace:
Insert(14)   # 14%10=4 → [4] empty → stored at [4]
Insert(24)   # 24%10=4 → [4] taken → probe [5] → empty → stored at [5]
Insert(34)   # 34%10=4 → [4],[5] taken → probe [6] → empty → stored at [6]
print(Search(34))  # → 6  (found at index 6)

How the 2D array works — HashTable[100][10]
Cambridge's exam structure — 2D array with Record class
# ── Record class ─────────────────────────────────────────────
class Record:
    def __init__(self, pKey, pData):
        self.Key = pKey     # integer
        self.Data = pData   # string

# ── Hash Table: 100 rows × 10 columns ────────────────────────
HashTable = []

def InitialiseHashTable():
    global HashTable
    HashTable = [[Record(-1, "")] * 10 for i in range(100)]
    # Creates 100 rows, each containing 10 Record objects.
    # ROW index  = the hash address (key % 100 tells you the row).
    # COLUMN index = position within that row.
    #   col[0] = first item stored at this address (no collision).
    #   col[1] = second item if there was a collision.
    #   col[2] = third item if there were two collisions. And so on.
    # Record(-1, "") signals an empty slot — Key=-1 means "nothing here".

def Hash(Key):
    return Key % 100   # returns 0–99

def InsertData(RecordData):
    global HashTable
    row = Hash(RecordData.Key)
    for col in range(10):
        if HashTable[row][col].Key == -1:   # empty column → insert here
            HashTable[row][col] = RecordData
            break

def GetRecord(Key):
    global HashTable
    row = Hash(Key)
    for col in range(10):
        if HashTable[row][col].Key == Key:
            return HashTable[row][col].Data   # ✅ found
    return "Not found"   # ← MUST be OUTSIDE the for loop

def ReadData():
    global HashTable
    File = open("HashTableData.txt")
    for Line in File:
        Data = Line.strip().split(",")   # strip() removes \n BEFORE split()
        InsertData(Record(int(Data[0]), Data[1]))
    File.close()

InitialiseHashTable()
ReadData()
for _ in range(5):
    key = int(input("Enter key: "))
    print(GetRecord(key))

⚠️ 6 marks students throw away on hash tables — memorise all of these
1. Empty sentinel = -1
→
Cambridge initialises hash tables with -1. The empty check is == -1 or .Key == -1. NOT None, NOT 0, NOT "". Use the exact sentinel the question specifies.
2. "Not found" outside the loop
→
In GetRecord(), return "Not found" must be AFTER the for loop ends — not inside it. Inside means you return it after checking just one slot. Cambridge puts 1 dedicated mark on this exact placement.
3. Always call Hash()
→
InsertData, GetRecord, and any delete function all start with row = Hash(Key). Never hardcode the row. Never write key % 100 directly in those functions — call Hash().
4. global HashTable
→
Every function that reads or modifies HashTable needs global HashTable. Without it, Python makes a local copy — your changes vanish. Same rule as Stack and Queue.
5. strip() before split()
→
File lines end with hidden \n. Use Line.strip().split(",") — strip() removes the newline FIRST, then split() by comma. Without strip(), your data string has a trailing newline. Cambridge checks this.
6. Know all 3 method names
→
Chaining = linked list at address. Overflow area = separate section. Neighbouring slots = probe forward. All three are valid ways to say the same methods — recognise them all.
✅ Final Summary — Everything in One Place

🔑 Hash table = skip the search, go directly to the data. address = key % size. One calculation. One jump. O(1). No loops.

💥 Collision = two keys calculate the same address. Expected. Always happens. You need one of three plans:

   🔗 Method 1 — Chaining: Each slot is the head of a linked list. Collision victim becomes a new node appended to that list. Search: hash → slot → walk list until found or null.

   🗃️ Method 2 — Overflow Area: Array split into main zone + overflow zone. Collision victim goes to overflow. Main slot stores a pointer there. Search: hash → main slot → follow pointer if needed.

   🔎 Method 3 — Neighbouring Slots: If home slot is full, check [home+1], [home+2]... Same array, no extras. Formula: (slot + 1) % size. Also called linear probing. Stop probing at an empty slot = "Not found".

🚨 The marks you keep losing: Empty check is .Key == -1. "Not found" is OUTSIDE the loop. Always write global HashTable. Always strip() before split(). Know all three method names. These five rules = guaranteed extra marks on every hash table question.
