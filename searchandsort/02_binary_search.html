What is Binary Search?
Divide and conquer — eliminate half the list every step
🧠 The analogy You're looking up "penguin" in a printed dictionary. You don't start at page 1. You open the middle — land on "M". "Penguin" comes after M, so you throw away the ENTIRE left half. Open the middle of the right half — land on "S". "Penguin" comes before S, throw away the right half. A few more cuts and you're there. That is binary search — you halve the problem with each step.
⚠️ The golden rule — never forget this Binary search ONLY works on a SORTED list. If the list is unsorted, binary search can give completely wrong answers. You must sort first, then search.
First (left boundary) Last (right boundary) Middle (checked this step) Found! Eliminated (not searched)

The Python Code 🐍
Every case from the four examples above is handled inside this single function
def binary_search(my_list, search_item):

    found         = False
    search_failed = False
    first         = 0
    last          = len(my_list) - 1

    while not found and not search_failed:

        middle = (first + last) // 2             # always the midpoint

        if my_list[middle] == search_item:
            found = True                        # exact match — done!

        elif first >= last:
            search_failed = True               # no more space to search

        elif my_list[middle] > search_item:
            last = middle - 1                  # value is in LEFT half

        else:
            first = middle + 1                 # value is in RIGHT half

    if found:
        print("Found at index:", middle)
        return middle
    else:
        print("Item not present in array")
        return -1
🔎 Every variable — what it does and why
found
→
Becomes TRUE when we find the exact match. Stops the loop.
search_failed
→
Becomes TRUE when first ≥ last AND no match found — the list has run out. Stops the loop.
first
→
Left boundary of the search zone. Starts at 0. Moves right when the answer is in the right half.
last
→
Right boundary of the search zone. Starts at len−1. Moves left when the answer is in the left half.
middle
→
(first + last) // 2 — always integer division. The element we check this step.
⚠️ Order of the elif branches — this matters! Cambridge expects this exact order:

1st: Check if list[middle] == search_item (exact match)
2nd: Check if first >= last (no space left — failed)
3rd: Check if list[middle] > search_item (go left: last = middle − 1)
4th (else): list[middle] must be less (go right: first = middle + 1)

If you put the "first >= last" check FIRST, you will fail to detect the final element before giving up.

B3 Write BinarySearch() — Cambridge Paper 4 style
Intermediate [6 marks] ▼ expand
A program stores sorted integers in the global 1D array DataStored. The global variable NumberItems stores how many elements are in the array.

The function BinarySearch() takes the integer parameter DataToFind, performs an iterative binary search on DataStored, and returns the index where DataToFind is found. If DataToFind is not found, the function returns −1.
Cambridge Paper 4 style — adapted from 9618/41/M/J/24. This is worth 6 marks. Write your full answer before checking.
  • Write program code for the iterative function BinarySearch(). [6]
Declare: found=False, search_failed=False, first=0, last=NumberItems−1. Loop: while not found and not search_failed. Inside loop: calculate middle = (first+last)//2, then four cases in order: (1) exact match → found=True, (2) first≥last → search_failed=True, (3) list[middle] > DataToFind → last=middle−1, (4) else → first=middle+1. After loop: return middle if found, else return −1. 
def BinarySearch(DataToFind):
    global DataStored, NumberItems

    found         = False
    search_failed = False
    first         = 0
    last          = NumberItems - 1

    while not found and not search_failed:

        middle = (first + last) // 2

        if DataStored[middle] == DataToFind:
            found = True

        elif first >= last:
            search_failed = True

        elif DataStored[middle] > DataToFind:
            last = middle - 1

        else:
            first = middle + 1

    if found:
        return middle
    else:
        return -1


# Mark scheme (6 marks):
# [1] found and search_failed declared and initialised correctly
# [1] first = 0, last = NumberItems - 1
# [1] while loop with both conditions (not found AND not search_failed)
# [1] middle = (first + last) // 2
# [1] all four cases handled in correct order
# [1] return middle if found, return -1 if not found
B4 Explain & Compare — Binary vs Linear
Hard [4 marks] ▼ expand
A sorted array contains 1024 elements. A programmer needs to search the array repeatedly for different values throughout the program.
  • State the maximum number of comparisons needed for a linear search and for a binary search on this array. [2]
  • Explain why binary search is more suitable for this scenario. Include reference to the precondition that must be met. [2]
Binary search halves the list each step. How many times can you halve 1024 before you reach 1? That's log₂(1024). What is log₂(1024)? 
# Maximum comparisons:

#   Linear search:  1024 comparisons
#   (worst case: value not found, or at the very last position)

#   Binary search:  log₂(1024) = 10 comparisons
#   (each step halves the remaining search space)
