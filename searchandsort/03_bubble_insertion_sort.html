Bubble Sort
Compare neighbours — swap if out of order — the largest value bubbles to the end each pass
🧠 The analogy Drop a heavy stone and a ping-pong ball in a bucket of water. The heavy stone sinks, the ping-pong ball floats to the top. Bubble sort does exactly this — the largest value "floats up" to the end of the array on each pass. After one full pass, the largest is in its final position. After two passes, the two largest are in place. Keep going until everything is sorted.

The Python Code — Bubble Sort 🐍
def bubble_sort(my_list):

    max_index = len(my_list) - 1
    n         = max_index - 1    # n shrinks each pass — avoids re-checking sorted end

    for i in range(max_index):    # outer: controls how many passes
        for j in range(n + 1):    # inner: compares pairs up to position n
            if my_list[j] > my_list[j + 1]:
                temp           = my_list[j]       # step 1: save j
                my_list[j]     = my_list[j + 1]  # step 2: overwrite j
                my_list[j + 1] = temp            # step 3: restore from temp
        n -= 1                    # last element is now sorted — shrink window
🔎 The three-step swap — why you need temp
temp = my_list[j]
→
Save a copy of the value before we overwrite it. Without this, it would be lost forever.
my_list[j] = my_list[j+1]
→
Copy the right value into the left position. The original left value is now gone — but we saved it in temp.
my_list[j+1] = temp
→
Place the saved left value into the right position. Swap complete.
n -= 1
→
After each full pass, the last element is in its correct final place. We don't need to compare it again — so we shrink n by 1.

03b
Insertion Sort
Pick one item at a time — slide it into the correct position on the sorted left side
🧠 The analogy You're sorting a hand of playing cards. You pick up the second card and check if it should go before or after the first — if before, slide the first one right to make room and drop the second card in. Then pick up the third card — find its correct place among the already-sorted cards, slide everything to the right, insert it. The left side is always sorted. The right side shrinks one card at a time.

The Python Code — Insertion Sort 🐍
def insertion_sort(my_list):

    n = len(my_list)

    for i in range(1, n):    # i = current element we're inserting

        value = my_list[i]       # save the value we're inserting
        j = i - 1               # j = index we're comparing with (starts one left)

        while j >= 0 and my_list[j] > value:
            my_list[j + 1] = my_list[j]  # shift right
            j -= 1                       # move left

        my_list[j + 1] = value          # insert into gap
🔎 Every line explained — in plain English
n = len(my_list)
→
How many items are in the list? We save that number as n. If the list is [5, 3, 1], then n = 3. We'll use n to control our loops — so we know where the list ends and don't go past it.
for i in range(1, n):
→
This is the outer loop — it picks one element at a time to insert into the sorted section.

Why start at 1 and not 0? Because index 0 (the first element) is already alone — a single element is trivially sorted. There's nothing to compare it with. So we start at the second element and ask: "where does this one belong?"

Think of it like sorting cards in your hand. You pick up the second card and decide where it goes relative to the first. Then the third card relative to the first two. And so on.
value = my_list[i]
→
We save a copy of the element we're about to insert.

Why? Because in the next step, we're going to shift elements to the right — and that will overwrite position i. If we didn't save the value first, it would be gone forever and we'd have nothing to insert.

Imagine you pick up a card from the table. You hold it in your hand while you make room for it. If you put it down before making room, it gets buried.
j = i - 1
→
j is our scanner — it looks at the sorted section (everything to the left of i) to find where our value belongs.

We start it at i - 1 because that's the element immediately to the left of the one we picked up — the last element of the already-sorted section. That's where we begin comparing.
while j >= 0
and my_list[j] > value:
→
This loop has two guards — both must be true to keep going. If either one fails, we stop.

Guard 1 : j >= 0: Don't walk off the left edge of the array. Index -1 doesn't exist. If j reaches -1, it means our value is smaller than everything in the sorted section — it belongs at the very beginning.

Guard 2 : my_list[j] > value: Is the element we're looking at bigger than our value? If yes, it needs to move right to make room. If no — it's smaller than or equal to our value, meaning our value belongs right after it. Stop here.
my_list[j + 1] = my_list[j]
→
The element at position j is too big — it needs to move one step to the right to make a gap for our value.

We write it into position j + 1 (one step right). This creates an empty slot at position j.

Think of it like sliding a book one spot to the right on a shelf to make room for a new one.

⚠️ Notice we're NOT using a temp variable here like in bubble sort. We don't need to — our value is already safely saved in value. We're just shifting, not swapping.
j -= 1
→
We just shifted the element at position j. Now we need to look at the next element to the left — is that one also too big? Does it also need to move?

j -= 1 moves our scanner one step left. Then the while loop checks again. We keep going until we find an element that's small enough — or until we fall off the left edge.
my_list[j + 1] = value
→
The while loop stopped. That means one of two things happened:

Case 1: j hit -1 — our value is the smallest thing seen so far. Insert at j + 1 = 0 — the very beginning. ✅

Case 2: my_list[j] is NOT bigger than our value — it's the "guardian". Our value goes right after the guardian, which is position j + 1. ✅

Either way, j + 1 is always the correct spot. We drop our saved value in.

🤔 Deep Dive: Why j -= 1 then j + 1?

❓ Wait — if we subtract 1 from j, then add 1 back… aren't we just going back to where we started? Isn't that pointless?

No — and here's exactly why.
The j -= 1 happens inside the loop, over and over again, while things are shifting.
The j + 1 happens after the loop ends, once, to drop the value in the right spot.

They're not cancelling each other — they happen at completely different moments, for completely different reasons.

🔍 So what is j -= 1 actually doing?

After we shift one element to the right, we need to look at the next element to the left to see if it also needs shifting. That's it. j -= 1 just moves our gaze one step left.

Think of it like scanning a bookshelf from right to left — after you move one book, you look at the next one. You don't stop; you keep going until you find a book that's already in the right place.

🔍 And why insert at j + 1 and not just j?

Because when the while loop stops, j is pointing at an element that is smaller than or equal to our value. That element stays where it is — it belongs there. Our value goes right after it, which is j + 1.

The loop stopped because j found a "guardian" — a number that says "you don't go before me." Our value goes immediately after the guardian.

🔍 But what if j goes all the way to -1? That's not even a real index!

It means our value is smaller than everything in the sorted section — it needs to go at index 0, the very beginning.

And j + 1 gives us exactly that: -1 + 1 = 0. The formula handles it perfectly without any special case. ✅

🎬 Example 1 — value ends up at the start: inserting 2 into [5, 8, _]

Start: value = 2, j = 1 (pointing at 8)
—— Loop iteration 1 ——
Is 8 > 2? YES → shift 8 right  →  array becomes [5, 8, 8]
j -= 1  →  j is now 0 (looking at 5)
—— Loop iteration 2 ——
Is 5 > 2? YES → shift 5 right  →  array becomes [5, 5, 8]
j -= 1  →  j is now -1
—— Loop ends (j < 0) ——
Insert at j + 1 = 0  →  array becomes [2, 5, 8] ✅
💡 j went: 1 → 0 → -1. It overshot past the beginning. j + 1 = 0 corrects it back to index 0.

🎬 Example 2 — value ends up in the middle: inserting 4 into [1, 3, 5, 7, _]

Start: value = 4, j = 3 (pointing at 7)
—— Loop iteration 1 ——
Is 7 > 4? YES → shift 7 right  →  array becomes [1, 3, 5, 7, 7]
j -= 1  →  j is now 2 (looking at 5)
—— Loop iteration 2 ——
Is 5 > 4? YES → shift 5 right  →  array becomes [1, 3, 5, 5, 7]
j -= 1  →  j is now 1 (looking at 3)
—— Loop iteration 3 ——
Is 3 > 4? NO → loop stops here. j stays at 1.
—— Insert ——
Insert at j + 1 = 2  →  array becomes [1, 3, 4, 5, 7] ✅
💡 j stopped at 1 (pointing at 3). 3 is the "guardian" — it's smaller than 4, so 4 goes after it → index 2 = j + 1.

📌 The one rule to remember

When the while loop ends, j is one step to the left of where our value belongs — either because it found a smaller element (the guardian), or because it fell off the edge (-1). So j + 1 is always exactly the right insertion spot. Every single time.

Bubble vs Insertion — What's the difference? ⚖️
Cambridge Paper 2 frequently asks you to compare these two algorithms
FeatureBubble SortInsertion Sort
Strategy Compare neighbours, swap if wrong order Pick one item, insert it into correct position in sorted section
Direction of movement Large values bubble rightward each pass Left side grows sorted, right side shrinks
Swap needed? Yes — 3-step swap using temp No — shifting (overwriting), then one final insert
Nearly sorted data Still does all passes regardless Much faster — inner while loop exits quickly
Worst-case comparisons O(n²) O(n²)
Best case (already sorted) O(n²) unless improved with flag O(n) — inner while never executes
Temp variable needed? Yes (for swap) Yes (saves item_to_be_inserted before shifting)

S2 Write BubbleSort() and InsertionSort() — global array style

A program stores integer numbers in the global 1D array DataStored.
The global variable NumberItems stores how many elements are in the array.

The program contains two sorting procedures: BubbleSort() and InsertionSort(). Both sort DataStored into ascending order.
Cambridge Paper 4 style — adapted from 9618/41/M/J/24 Q1(d) and 9618/41/O/N/23 Q2.
In the exam you will write one sorting procedure. Here you practice both so you are ready for either.
  • Write program code for the procedure BubbleSort(). [4]
  • Write program code for the procedure InsertionSort(). [4]
BubbleSort hint: declare global DataStored, NumberItems. max_index = NumberItems − 1. n = max_index − 1. Outer loop: range(max_index). Inner loop: range(n + 1). If DataStored[j] > DataStored[j+1] → 3-step swap using temp. After inner loop: n -= 1.

InsertionSort hint: declare global DataStored, NumberItems. Outer loop: range(1, NumberItems). Save value = DataStored[i]. Set current = i − 1. While current >= 0 AND DataStored[current] > value: shift right, current -= 1. After while: DataStored[current + 1] = value. 
# ── BubbleSort ──────────────────────────────────────────────────

def BubbleSort():
    global DataStored, NumberItems

    max_index = NumberItems - 1
    n         = max_index - 1

    for i in range(max_index):
        for j in range(n + 1):
            if DataStored[j] > DataStored[j + 1]:
                temp                 = DataStored[j]
                DataStored[j]        = DataStored[j + 1]
                DataStored[j + 1]  = temp
        n -= 1


# Mark scheme BubbleSort (4 marks):
# [1] outer loop — correct range (max_index passes)
# [1] inner loop — compares up to n, not the full array
# [1] correct 3-step swap using a temp variable
# [1] n decremented after each pass


# ── InsertionSort ───────────────────────────────────────────────

def InsertionSort():
    global DataStored, NumberItems

    for i in range(1, NumberItems):
        value   = DataStored[i]
        current = i - 1

        while current >= 0 and DataStored[current] > value:
            DataStored[current + 1] = DataStored[current]
            current -= 1

        DataStored[current + 1] = value


# Mark scheme InsertionSort (4 marks):
# [1] outer loop starting from index 1
# [1] value saved before shifting begins
# [1] while loop with both conditions (current >= 0 AND comparison)
# [1] correct shift and final insertion at current + 1
S3 Write Bubble() — function that takes and returns an array
Hard [7 marks] ▼ expand
A program sorts string data using different sorting methods.

The function CompareStrings() takes two strings as parameters. It compares each string one character at a time to identify which string comes first alphabetically.

The function:
  • returns 1 if the first parameter comes before the second alphabetically
  • returns 2 if the second parameter comes before the first alphabetically

Assume all strings are lower case and a difference will always be found before the end of a string is reached. Do not use a built-in string comparison function — compare one character at a time.
Cambridge Paper 4 style — directly from 9618/41/O/N/24 Q1(c) and Q1(d).
This is the hardest sorting question Cambridge has set. It tests two things at once: writing a character-by-character comparator, then using it inside bubble sort.
  • Write program code for CompareStrings(). The strings must be compared one character at a time. Do not use any built-in string comparison. [4]
  • The function Bubble() takes an array of strings as a parameter, sorts the data into ascending alphabetical order using bubble sort, and returns the sorted array. The bubble sort must use CompareStrings() to compare each pair of strings.
    Write program code for Bubble(). [3]
CompareStrings hint: Use a loop with index starting at 0. Each iteration get the character at that index from both strings using indexing (string[i]). Compare using ord() or directly with < / >. As soon as the characters differ, return 1 or 2. Keep going while the characters are equal.

Bubble hint: The structure is almost identical to a normal bubble sort, but instead of if my_list[j] > my_list[j+1], you write if CompareStrings(my_list[j], my_list[j+1]) == 2. This means "the second string comes first alphabetically — so swap them." Still use a 3-step swap with temp. Return the array at the end. 
# ── CompareStrings ──────────────────────────────────────────────

def CompareStrings(str1, str2):
    index = 0
    while str1[index] == str2[index]:
        index += 1
    if str1[index] < str2[index]:
        return 1    # str1 comes first alphabetically
    else:
        return 2    # str2 comes first alphabetically


# Mark scheme CompareStrings (4 marks):
# [1] loop that iterates character by character (not using built-in compare)
# [1] correct character access: str1[index], str2[index]
# [1] loop condition: continues while characters are equal
# [1] correct return values: 1 if str1 first, 2 if str2 first


# ── Bubble ──────────────────────────────────────────────────────

def Bubble(my_list):
    max_index = len(my_list) - 1
    n         = max_index - 1

    for i in range(max_index):
        for j in range(n + 1):
            if CompareStrings(my_list[j], my_list[j + 1]) == 2:
                temp           = my_list[j]
                my_list[j]     = my_list[j + 1]
                my_list[j + 1] = temp
        n -= 1

    return my_list

# [1] correct 3-step swap using temp
# [1] returns the sorted array
