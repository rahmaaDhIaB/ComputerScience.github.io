Stacks — LIFO (Last In, First Out)
The Pringles can of data structures 🍟
🍟 The Pringles analogy — this is literally it You stack chips into a Pringles can. The LAST chip you put in sits at the TOP. When you reach in, you grab the TOP one first. You physically cannot get to the bottom ones without removing the top ones first. That's a stack — Last In, First Out (LIFO).
🌐
Real World
Browser back button
Every page you visit gets pushed onto a stack. Hit back? You pop the most recent page off and go to the one underneath. That's literally a stack in your browser. It's giving LIFO fr fr.
↩️
Gaming / Editors
Ctrl+Z — Undo
Every action you do (draw a line, type a word) gets pushed onto the undo stack. Hit Ctrl+Z? It pops the last action off and reverses it. No cap, stacks are why undo exists.
📞
WhatsApp
Call stack (actual coding)
When a function calls another function, Python pushes the new one onto the call stack. When it returns, it pops back to where it was. Your entire Python program runs on a stack under the hood.
🍽️
Kitchen Vibes
Stack of plates
Canteen puts clean plates on top. You take from the top. The plate at the bottom hasn't moved in years. Classic LIFO. The bottom plate is lowkey suffering.
Stack operations — know every single one 🧊
OperationWhat it doesCambridge saysAnalogy
Push(x) Add element x to the top of the stack. Update TopOfStack first, then insert. Check full first → if full return FALSE Put a chip into the Pringles can
Pop() Remove and return the top element. Decrease TopOfStack by 1. Check empty first → if empty return -999 Take the top chip out
Peek() / Top() Look at the top element WITHOUT removing it. Return Stack[TopOfStack] without changing pointer Look at top chip without eating it
How a stack looks in memory 🧠
The array is fixed size. TopOfStack pointer tells us where the top is. Starts at -1 (empty).
🧠 The pointer logic — don't get this wrong
TopOfStack = -1
→
Stack is empty. Index -1 means "nothing here yet". This is the initialisation value — always.
Push: increment FIRST
→
TopOfStack += 1 then Stack[TopOfStack] = value. Move the pointer up, THEN insert.
Pop: read FIRST
→
val = Stack[TopOfStack] then TopOfStack -= 1. Read the value, THEN move pointer down.
Full check
→
TopOfStack == SIZE - 1. Stack of 30 → full when TopOfStack == 29.
Stack — complete Python code 🐍
This is the Cambridge-approved template. Learn every line.
# ── STACK DECLARATION ────────────────────────────────────────
SIZE = 30
Stack = [None] * SIZE   # 1D array of 30 elements, initialised to None
TopOfStack = -1         # global pointer, -1 = stack is empty


# ── PUSH ─────────────────────────────────────────────────────
def Push(value):
    global Stack, TopOfStack   # ⚠️ MUST use global to modify these
    if TopOfStack == SIZE - 1:  # stack is full
        return False
    else:
        TopOfStack += 1
        Stack[TopOfStack] = value
        return True


# ── POP ──────────────────────────────────────────────────────
def Pop():
    global Stack, TopOfStack
    if TopOfStack == -1:   # stack is empty
        return -999        # Cambridge sentinel value for "empty"
    else:
        val = Stack[TopOfStack]
        TopOfStack -= 1
        return val
⚠️ COMMON MISTAKE — global keyword In Python, if you modify a global variable inside a function WITHOUT writing global, Python creates a NEW local variable instead. Your Stack and TopOfStack won't actually change. Cambridge will deduct marks for this. Write global Stack, TopOfStack at the top of every function that modifies them. No exceptions.

Queue operations — both pointers, always 🧊
OperationWhat it doesCheck firstPointer update
Enqueue(x) Add element to the REAR of the queue Is TailPointer == SIZE? If yes → full Queue[TailPointer] = x then TailPointer += 1
Dequeue() Remove and return element from the FRONT Is HeadPointer == -1? If yes → empty val = Queue[HeadPointer] then HeadPointer += 1
How a linear queue looks in memory 🧠
HeadPointer = -1 (Cambridge init). TailPointer = 0. Head tracks front (remove), Tail tracks next insert position.
⚠️ The BIG problem with linear queues See index [0] up there? It's empty — wasted. Once dequeued items leave the front, those spaces can NEVER be reused. The queue slowly slides to the right until TailPointer hits the end — and the queue looks "full" even when half the array is empty. This is why we need circular queues. 
# ── LINEAR QUEUE DECLARATION ─────────────────────────────────
SIZE = 50
Queue = [None] * SIZE
HeadPointer = -1    # Cambridge init = -1 (empty)
TailPointer = 0     # Cambridge init = 0 (next insert at index 0)


# ── ENQUEUE ──────────────────────────────────────────────────
def Enqueue(value):
    global Queue, HeadPointer, TailPointer
    if TailPointer == SIZE:      # queue is full
        print("Queue full")
    else:
        if HeadPointer == -1:   # first item ever inserted
            HeadPointer = 0
        Queue[TailPointer] = value
        TailPointer += 1


# ── DEQUEUE ──────────────────────────────────────────────────
def Dequeue():
    global Queue, HeadPointer, TailPointer
    if HeadPointer == -1 or HeadPointer == TailPointer:
        print("Queue empty")
        return "Empty"   # Cambridge sentinel value
    else:
        val = Queue[HeadPointer]
        HeadPointer += 1
        return val
03
Circular Queue — what Cambridge really tests 💀
Wrap-around with modulo — the upgrade that fixes linear queue's problem
🔄 The circular buffer idea — genius tbh Instead of letting the queue slide to the right forever, we wrap around. When the tail pointer reaches the end of the array, it wraps back to index 0 — IF there's space there from previously dequeued items. It's like a roundabout. No wasted space. No cap, this is the one Cambridge loves to test.
How a circular queue wraps around 🔄
🔑 The modulo wrap — one formula to rule them all
(TailPointer+1) % SIZE
→
After inserting, move tail forward. If tail was at index 7 (last) in size-8 array: (7+1)%8 = 0. It wraps back to the start! That's the magic.
(HeadPointer+1) % SIZE
→
Same logic for head when dequeuing. Head wraps around too. Both pointers use the same formula.
NumberItems == 0
→
Empty check. Circular queues use a counter — not pointer comparison — because the pointer positions alone are ambiguous.
NumberItems == SIZE
→
Full check. When counter equals array size, every slot is occupied.
# ── CIRCULAR QUEUE DECLARATION ───────────────────────────────
SIZE = 20
CircQueue = [-1] * SIZE   # initialised with -1 (Cambridge style)
HeadPointer = -1          # both pointers start at -1
TailPointer = -1
NumberItems = 0           # counter tracks how many items are stored


# ── ENQUEUE (circular) ───────────────────────────────────────
def Enqueue(value):
    global CircQueue, HeadPointer, TailPointer, NumberItems
    if NumberItems == SIZE:    # queue is full
        return False
    else:
        TailPointer = (TailPointer + 1) % SIZE   # wrap-around ✅
        CircQueue[TailPointer] = value
        if NumberItems == 0:             # first item — set head too
            HeadPointer = TailPointer
        NumberItems += 1
        return True


# ── DEQUEUE (circular) ───────────────────────────────────────
def Dequeue():
    global CircQueue, HeadPointer, TailPointer, NumberItems
    if NumberItems == 0:     # queue is empty
        return -1           # Cambridge sentinel for circular = -1
    else:
        val = CircQueue[HeadPointer]
        NumberItems -= 1
        if NumberItems == 0:  # 🔑 last item just removed — reset BOTH pointers
            HeadPointer = -1
            TailPointer = -1
        else:
            HeadPointer = (HeadPointer + 1) % SIZE  # wrap-around ✅
        return val
⚠️ MOST MISSED MARK — reset pointers when queue empties When the last item is dequeued (NumberItems becomes 0), you MUST reset both HeadPointer = -1 and TailPointer = -1. Cambridge specifically tests this. Students who forget this lose a mark every single time. It's giving free marks to whoever remembers. Don't be the one who forgets.

E1 Game Score History — Stack implementation
Easy [10 marks] ▼ expand
A computer game stores the player's previous scores in a stack. The stack is represented as a 1D array of 30 elements with the identifier Scores. The global pointer TopOfStack stores the index of the last element inserted. TopOfStack is initialised to –1.
  • Write program code to declare Scores, initialise each element to a null value, and declare/initialise TopOfStack. [2]
  • The function AddScore() takes an integer parameter. If the stack is full, it returns FALSE. Otherwise, it inserts the parameter into the stack, updates the pointer, and returns TRUE. Write AddScore(). [4]
  • The function UndoScore() returns the most recent score and removes it from the stack. If the stack is empty, it returns -999. Write UndoScore(). [4]
Cambridge mark scheme points: [2] declaration: array of correct size + None, TopOfStack = -1. [4] for AddScore: checks full (TopOfStack == 29), increments, assigns, returns True/False. [4] for UndoScore: checks empty (TopOfStack == -1), reads value, decrements, returns -999 on empty.
Declaration: Scores = [None] * 30 and TopOfStack = -1. AddScore: full when TopOfStack == 29. Increment FIRST then assign. UndoScore: empty when TopOfStack == -1. Read value FIRST then decrement. Don't forget global keyword! 
# ── PART (a) — Declaration ───────────────────────────────────
SIZE = 30
Scores = [None] * SIZE
TopOfStack = -1

# Mark scheme [2]:
# [1] Scores declared as 1D array of 30 elements initialised to None/null
# [1] TopOfStack declared and initialised to -1


# ── PART (b) — AddScore() (Push) ─────────────────────────────
def AddScore(value):
    global Scores, TopOfStack
    if TopOfStack == SIZE - 1:   # full check
        return False
    else:
        TopOfStack += 1
        Scores[TopOfStack] = value
        return True

# Mark scheme [4]:
# [1] Checks if stack is full: TopOfStack == SIZE-1 (or == 29)
# [1] Returns False if full
# [1] Increments TopOfStack and inserts value at Scores[TopOfStack]
# [1] Returns True if successfully inserted


# ── PART (c) — UndoScore() (Pop) ────────────────────────────
def UndoScore():
    global Scores, TopOfStack
    if TopOfStack == -1:     # empty check
        return -999
    else:
        val = Scores[TopOfStack]
        TopOfStack -= 1
        return val

# Mark scheme [4]:
# [1] Checks if stack is empty: TopOfStack == -1
# [1] Returns -999 if empty
# [1] Reads value at Scores[TopOfStack]
# [1] Decrements TopOfStack and returns value

E2 High Score Tracker — Random scores + Find best/worst
Intermediate [9 marks] ▼ expand
Continuing from E1. The game generates 40 random scores between 0 and 1000 (inclusive) and attempts to insert each one using AddScore(). If the return value indicates the stack is full, output "Score stack full" and stop. Then write AnalyseScores() that pops all scores and outputs the highest and lowest scores ever achieved.
  • Write the main program — 40 random scores, AddScore() each one, stop + output "Score stack full" if full. [4]
  • Write AnalyseScores() — pops until empty, outputs highest and lowest with messages. [4]
  • Extend main program to call AnalyseScores(). [1]
Key insight: Use the FIRST popped value to initialise both highest and lowest. Pop returns -999 when empty — use that as loop termination. Cambridge checks that initial values come from actual data, not arbitrary numbers.
Main: import random, random.randint(0, 1000). Loop 40 times. Call AddScore() — if returns False, print message and break. AnalyseScores: first_val = UndoScore() to initialise. Then loop: val = UndoScore(), while val != -999, update highest/lowest. 
import random

# ── PART (d) — Main program ───────────────────────────────────
for i in range(40):
    score = random.randint(0, 1000)
    result = AddScore(score)
    if result == False:
        print("Score stack full")
        break

# Mark scheme [4]:
# [1] Loop runs up to 40 times
# [1] Generates random integer 0 to 1000 inclusive
# [1] Calls AddScore() with generated integer
# [1] Checks return value — outputs message and stops if False


# ── PART (e) — AnalyseScores() ───────────────────────────────
def AnalyseScores():
    first_val = UndoScore()
    if first_val == -999:
        print("No scores recorded")
        return
    highest = first_val
    lowest = first_val

    val = UndoScore()
    while val != -999:
        if val > highest:
            highest = val
        if val < lowest:
            lowest = val
        val = UndoScore()

    print("Highest score achieved:", highest)
    print("Lowest score achieved:", lowest)

# Mark scheme [4]:
# [1] Pops all integers until stack empty (terminates on -999)
# [1] Correctly tracks highest value
# [1] Correctly tracks lowest value
# [1] Outputs both values with appropriate messages

AnalyseScores()

E3 Game ID Queue — Linear queue for shop
Easy [6 marks] ▼ expand
A shop sells computer games. Each game has a unique identifier (ID) of string data type. The shop uses a linear queue to process game IDs waiting to be shipped. The queue is implemented using a 1D array GameQueue with 50 elements of type string. HeadPointer = -1, TailPointer = 0.
  • The procedure AddToQueue() takes a string parameter. If the queue is full, it outputs "Queue is full". If not full, it inserts the parameter and updates the relevant pointer(s). Write AddToQueue(). [3]
  • The function RemoveFromQueue() checks if the queue is empty. If empty, it outputs "Queue is empty" and returns "Empty". If not empty, it returns the first element and updates the relevant pointer(s). Write RemoveFromQueue(). [3]
Linear queue quirk: HeadPointer starts at -1 but is set to 0 on the FIRST enqueue. TailPointer starts at 0 — the index of the next insert. Full when TailPointer == SIZE (50).
AddToQueue: check TailPointer == SIZE (full). If first insert, set HeadPointer = 0. Insert at Queue[TailPointer], then TailPointer++. RemoveFromQueue: check HeadPointer == -1 or HeadPointer == TailPointer (empty). Read value, increment HeadPointer, return value. 
SIZE = 50
GameQueue = [None] * SIZE
HeadPointer = -1
TailPointer = 0


def AddToQueue(gameID):
    global GameQueue, HeadPointer, TailPointer
    if TailPointer == SIZE:
        print("Queue is full")
    else:
        if HeadPointer == -1:
            HeadPointer = 0
        GameQueue[TailPointer] = gameID
        TailPointer += 1

# Mark scheme [3]:
# [1] Checks full (TailPointer == SIZE) and outputs "Queue is full"
# [1] Inserts value at GameQueue[TailPointer]
# [1] Updates TailPointer (increment), sets HeadPointer=0 on first insert


def RemoveFromQueue():
    global GameQueue, HeadPointer, TailPointer
    if HeadPointer == -1 or HeadPointer == TailPointer:
        print("Queue is empty")
        return "Empty"
    else:
        val = GameQueue[HeadPointer]
        HeadPointer += 1
        return val

# Mark scheme [3]:
# [1] Checks empty condition and outputs "Queue is empty"
# [1] Returns "Empty" string on empty
# [1] Returns value and increments HeadPointer

E4 Customer Call Queue — Circular queue for call centre
Intermediate [8 marks] ▼ expand
A call centre uses a circular queue to manage customer calls. The queue uses a 1D array of 20 integers (customer IDs) initialised with -1. HeadPointer = -1, TailPointer = -1, NumberItems = 0. Write the functions for adding and removing customers.
  • The function JoinQueue() takes a customer ID and adds it to the queue. Returns TRUE if successful, FALSE if full. [4]
  • The function ServeCustomer() removes and returns the next customer ID. Returns -1 if the queue is empty. After serving the last customer, reset both pointers to -1. [4]
The mark Cambridge loves to test: When NumberItems becomes 0 after a dequeue, reset HeadPointer = -1 and TailPointer = -1. This is worth 1 dedicated mark. Don't forget it.
JoinQueue: full = NumberItems == SIZE. TailPointer = (TailPointer + 1) % SIZE. If NumberItems was 0, set HeadPointer = TailPointer. Increment NumberItems. ServeCustomer: empty = NumberItems == 0, return -1. Read value. Decrement NumberItems. If NumberItems == 0 → reset both to -1. Else HeadPointer = (HeadPointer+1) % SIZE. 
SIZE = 20
CallQueue = [-1] * SIZE
HeadPointer = -1
TailPointer = -1
NumberItems = 0


def JoinQueue(customerID):
    global CallQueue, HeadPointer, TailPointer, NumberItems
    if NumberItems == SIZE:
        return False
    TailPointer = (TailPointer + 1) % SIZE
    CallQueue[TailPointer] = customerID
    if NumberItems == 0:
        HeadPointer = TailPointer
    NumberItems += 1
    return True

# Mark scheme [4]:
# [1] Checks full: NumberItems == SIZE, returns False
# [1] TailPointer updated with modulo: (TailPointer+1) % SIZE
# [1] Customer ID stored at CallQueue[TailPointer]
# [1] NumberItems incremented; HeadPointer set on first insert; returns True


def ServeCustomer():
    global CallQueue, HeadPointer, TailPointer, NumberItems
    if NumberItems == 0:
        return -1
    customerID = CallQueue[HeadPointer]
    NumberItems -= 1
    if NumberItems == 0:
        HeadPointer = -1
        TailPointer = -1
    else:
        HeadPointer = (HeadPointer + 1) % SIZE
    return customerID

# Mark scheme [4]:
# [1] Checks empty: NumberItems == 0, returns -1
# [1] Returns value at CallQueue[HeadPointer]
# [1] HeadPointer updated with modulo: (HeadPointer+1) % SIZE
# [1] When NumberItems becomes 0: resets HeadPointer AND TailPointer to -1  ← most missed!

E5 Product Code Validator — Check digit + queue storage
Hard [10 marks] ▼ expand
A warehouse processes product codes. Each code is 7 characters: first 6 are digits, the 7th is a check digit. The check digit is calculated by:
  • multiply digits at positions 0, 2, 4 by 1
  • multiply digits at positions 1, 3, 5 by 3
  • sum all products, divide by 10 using integer division → check digit
  • if check digit = 10, it becomes 'X'
Example — "9541234": 9×1=9, 5×3=15, 4×1=4, 1×3=3, 2×1=2, 3×3=9 → sum=42 → 42//10=4 → s[6]='4' → VALID ✅
  • Write ValidateProduct(s) — takes a 7-char string, returns True if valid, False if not. [4]
  • Write ProcessProducts() — takes 10 inputs, validates each using check digit, stores the FIRST 6 characters in a queue (using AddToQueue()) if valid. Count and output number of invalid items. [6]
Check digit formula recap: weights = [1, 3, 1, 3, 1, 3] for positions 0–5. Sum all (int(s[i]) × weights[i]). check = sum // 10. If check == 10 → expected = 'X'. Else expected = str(check). Valid if s[6] == expected.
Validate: weights = [1,3,1,3,1,3]. Loop positions 0-5. total += int(s[i]) * weights[i]. check = total // 10. If check == 10, expected = 'X', else expected = str(check). Return s[6] == expected. ProcessProducts: loop 10 times, input. Call ValidateProduct. If valid → AddToQueue(s[:6]). Else invalid_count++. Print count at end. 
# ── ValidateProduct() ────────────────────────────────────────
def ValidateProduct(code):
    weights = [1, 3, 1, 3, 1, 3]
    total = 0
    for i in range(6):
        total += int(code[i]) * weights[i]
    check = total // 10
    if check == 10:
        expected = 'X'
    else:
        expected = str(check)
    return code[6] == expected

# Mark scheme [4]:
# [1] Multiplies positions 0,2,4 by 1 and 1,3,5 by 3
# [1] Sums all products correctly
# [1] Calculates check digit using integer division by 10
# [1] Handles both cases: == 10 → 'X', else str digit; compares to s[6]


# ── ProcessProducts() ────────────────────────────────────────
def ProcessProducts():
    invalid_count = 0
    for i in range(10):
        product = input("Enter 7-character product code: ")
        if ValidateProduct(product):
            AddToQueue(product[:6])   # store first 6 characters only
        else:
            invalid_count += 1
    print("Number of invalid products:", invalid_count)

# Mark scheme [6]:
# [1] Takes exactly 10 inputs from user
# [1] Calls ValidateProduct() on each input
# [1] Calls AddToQueue() with correct value if valid
# [1] Stores only first 6 characters (product[:6])
# [1] Counts invalid items correctly
# [1] Outputs number of invalid items at the end

Examiner's Secrets 🕵️ — what they DON'T tell you
Marks students lose every year. Don't be that student.
⚠️ 6 marks students throw away every year — memorise these
1. Pointer initialisation
→
They ALWAYS test pointer init. Stack: TopOfStack = -1. Linear queue: HeadPointer = -1, TailPointer = 0. Circular: both -1, NumberItems = 0. Get this wrong = lose a mark instantly.
2. Circular pointer reset
→
When NumberItems becomes 0 after Dequeue: reset both pointers to -1. This is a dedicated mark students miss every exam. No cap.
3. global keyword in Python
→
Modifying global arrays/pointers inside functions REQUIRES global. Without it, Python makes a local copy. Your changes won't persist. Deduct mark.
4. Always check before operating
→
No check for empty before Pop/Dequeue = lose a mark. No check for full before Push/Enqueue = lose a mark. Both checks are always tested.
5. Sentinel return values matter
→
Stack empty → -999. Linear queue empty → "Empty". Circular queue empty → -1. Wrong sentinel = wrong mark. Read the question.
6. Return type must match
→
If question says "function returns Boolean" — your function MUST have return True / return False. If it says "procedure" — no return needed. Cambridge is strict on this.
✅ Vibe Check — Key Takeaways (fr fr no cap)

🍟 Stack = LIFO. Last in, first out. Pringles can. Push goes up, Pop comes down. TopOfStack = -1 to start.

🍔 Linear Queue = FIFO. First in, first out. McDonald's drive-thru. Enqueue at rear, Dequeue at front. Problem: wasted space.

🔄 Circular Queue = FIFO with modulo. (pointer + 1) % SIZE is the spell that wraps around. NumberItems counter for empty/full. Reset both pointers when it empties. Bet.

🚨 Cambridge mark scheme traps: Always check before operating. Always use global. Always reset circular pointers on empty. Always match the return type. These four things are worth at least 4 marks on every paper. It's giving free marks if you remember them.
