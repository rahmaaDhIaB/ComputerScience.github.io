Linear Search
Check every element one by one — left to right
✅ Works on Sorted Unsorted — works on both!
⏱ Worst case n comparisons — must check every element
🏁 Stops when found = TRUE or index reaches max_index
📍 index starts at −1 (so first increment → index 0)

def linear_search(my_list, search_value):
    max_index = len(my_list) - 1
    found     = False
    index     = -1          # ← starts at -1

    while not found and index < max_index:
        index += 1           # ← increment FIRST
        if my_list[index] == search_value:
            found = True

    if found:
        print("Found at:", index)
    else:
        print("Not found")
⚠️ Don't forget: index += 1 happens INSIDE the loop, BEFORE the if-check.
🎤 MR. RAYMOND'S SCRIPT (Linear Search):
“Look at the `index` variable. This is the biggest trap. It starts at -1, not 0. Why? Because the first thing we do inside the loop is `index += 1`. If we started at 0, we would skip element 0.
Notice the `while` condition: `not found AND index < max_index`. If we reach the end, `index` becomes equal to `max_index`, the loop stops, and we output 'Not found'.
Exam question: 'Why start at -1?' — To allow the first increment to hit index 0. 'Why two conditions?' — Efficiency. Stop as soon as we find it, don't keep searching.”
Walk through the visual array [3,7,1,9,4] searching for 9. Point at the boxes. Show them the 'dead' boxes vs the 'found' box.

Binary Search
Halve the search space every step
⚠️ Requires SORTED list only — won't work unsorted!
⏱ Worst case log₂(n) — 1024 items = only 10 steps!
🏁 Stops when found = TRUE or search_failed = TRUE
🔢 Middle (first + last) // 2 — integer division

def binary_search(my_list, search_item):
    found = search_failed = False
    first = 0;  last = len(my_list) - 1

    while not found and not search_failed:
        middle = (first + last) // 2

        if   my_list[middle] == search_item: found = True
        elif first >= last:                  search_failed = True
        elif my_list[middle] > search_item:  last  = middle - 1
        else:                                first = middle + 1

    if found: print("Found at:", middle)
    else:     print("Not present")
⚠️ Order matters! Check exact match → then first≥last → then go left → then go right. Cambridge marks this order!
🎤 MR. RAYMOND'S SCRIPT (Binary Search):
“Binary Search is ruthless. It demands a sorted list. If the data isn't sorted, this algorithm *will* fail.
Watch the order of the `if` statements: 1) Did we find it? 2) Is the search space empty? 3) Is the middle value *greater* than what we want? (Go left) 4) Else, go right.
Key variable: `middle = (first + last) // 2`. Use integer division. For 7 elements, the first middle is index 3. For 6 elements, it's index 2.
Exam trick: They will ask 'What is the value of `search_failed` after iteration 2?' You must trace `first` and `last` moving closer together until they cross.”
Draw a number line on the board. Show them how 1024 items become 512, then 256, then 128... down to 1. That's 10 steps.
🔢
Binary Search — Variables Explained
Know every variable. Examiners ask about them.
first
Left boundary. Starts at 0. Moves RIGHT (+1 past middle) when value is in right half.
last
Right boundary. Starts at len−1. Moves LEFT (−1 past middle) when value is in left half.
middle
(first + last) // 2. The index we check each step. Recalculated every iteration.
search_failed
Becomes TRUE when first ≥ last and no match found. Means: no space left to search.
🧠 When does search_failed become TRUE?
When first ≥ last — meaning only 1 element is left and it's not the match. No more halves to check. Loop exits, output "not found."

Bubble Sort
Compare neighbours. Swap. Largest bubbles to the end.
💡 Idea Each pass, biggest unsorted value floats to its final spot
🔄 Passes n−1 passes total (n = number of elements)
📦 Swap needs temp variable — 3 step swap
📉 n shrinks After each pass: n -= 1 (sorted end is off-limits)

def BubbleSort():
    global DataStored, NumberItems
    max_index = NumberItems - 1
    n         = max_index - 1   # shrinks each pass

    for i in range(max_index):
        for j in range(n + 1):
            if DataStored[j] > DataStored[j+1]:
                temp              = DataStored[j]      # step 1
                DataStored[j]     = DataStored[j+1]   # step 2
                DataStored[j+1]  = temp               # step 3
        n -= 1  # ← OUTSIDE inner loop, INSIDE outer
⚠️ n -= 1 position matters! It must be OUTSIDE the inner loop but INSIDE the outer loop.
🎤 MR. RAYMOND'S SCRIPT (Bubble Sort):
“Watch the animation. Compare j and j+1. If the left is bigger than the right, swap them. The biggest number 'bubbles' up to the end.
The variable `n` shrinks (`n -= 1`) after each full pass. Why? Because the last element is already sorted, we don't need to check it again.
The swap requires 3 lines:
1. `temp = DataStored[j]`
2. `DataStored[j] = DataStored[j+1]`
3. `DataStored[j+1] = temp`
If you forget `temp`, you lose the value. I will deduct marks for that in the exam.”
Use two students holding cards. Ask them to swap. If they don't use a 'temp' table, they can't do it. This visual sticks.

def InsertionSort():
    global DataStored, NumberItems

    for i in range(1, NumberItems):    # start at 1!
        value   = DataStored[i]          # save the item
        current = i - 1               # look left

        while current >= 0 and DataStored[current] > value:
            DataStored[current+1] = DataStored[current]  # shift right
            current -= 1                                 # move left

        DataStored[current+1] = value  # insert here
⚠️ Always insert at current + 1! When loop ends, current overshot one step left. current + 1 is the gap.
🎤 MR. RAYMOND'S SCRIPT (Insertion Sort):
“Imagine you have a deck of cards in your left hand (sorted) and you pick a new card from the table with your right hand (`value`). You slide the bigger cards to the right to make space.
We start at `i = 1` because index 0 alone is considered 'already sorted'.
The tricky part: The `while` loop shifts items right. When the loop stops, `current` is pointing to the last item that was *smaller* than `value`. So the empty spot is at `current + 1`.
Common exam error: Students write `DataStored[current] = value` instead of `DataStored[current+1]`. Don't do that.”
Act it out. Hold up your hand like it's holding sorted cards. Use the other hand to 'insert' a card. Show the shifting motion.
