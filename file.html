Records
Grouping related data under one name — like a row in a spreadsheet
The F1 analogy Imagine a spreadsheet of F1 teams. Each row is one team: position, team name, two drivers, points. All different data types but they belong together — that's a record. Instead of 5 separate variables floating around, you bundle them into one named thing.
Think of it like a spreadsheet row
Position TeamName Driver_1 Driver_2 Points
1 Red Bull Racing Verstappen Perez 860
2 Mercedes Hamilton Russell 409
3 Ferrari Leclerc Sainz 406
Each row above = one record instance. The column headers = the record type (the template).
Declaring a record in Python
Python has no built-in "record" keyword. We fake it with a class. But it's a SIMPLE class — no private attributes, no getters, no setters. Just a container.
# ── DECLARE THE RECORD TYPE ────────────────────────────────
class TeamRecord:
    def __init__(self, position, team_name, driver_1, driver_2, points):
        self.Position = position      # INTEGER  ← NO __ before name = PUBLIC
        self.TeamName = team_name     # STRING   ← NO __ before name = PUBLIC
        self.Driver_1 = driver_1      # STRING
        self.Driver_2 = driver_2      # STRING
        self.Points   = points        # INTEGER
Every line broken down
class TeamRecord:
→
Creates the record type — the template. No real data yet, just the structure (the column headers).
def __init__(self, ...)
→
The constructor. Runs automatically when you create a new record. All the field names are listed as parameters here.
self.TeamName = team_name
→
Creates a field called TeamName inside this record and stores the value. No __ = public — anyone can access it directly.
Instantiating a record
Creating actual record instances with real values.
# ── CREATE RECORD INSTANCES ────────────────────────────────
team1 = TeamRecord(1, "Red Bull Racing", "Verstappen", "Perez", 860)
team2 = TeamRecord(2, "Mercedes",       "Hamilton",  "Russell", 409)

# ── ACCESS FIELDS DIRECTLY — no getter needed ──────────────
print(team1.TeamName)    # output: Red Bull Racing
print(team1.Points)      # output: 860
team1.Points = 900        # change it directly — also works!

# ── STORE MANY RECORDS IN A LIST ───────────────────────────
teams = [team1, team2]
for team in teams:
    print(team.Position, team.TeamName, team.Points)
Your question: "Is a record the same as an OOP class?"
YES — the Python syntax is almost identical. The only difference is one character: __
RECORD — public fields
class TeamRecord:
  def __init__(self, name, pts):
    self.Name   = name
    self.Points = pts
self.Name — NO double underscore
This means the field is PUBLIC.

From outside the class you can:
print(t.Name) — read it
t.Points = 900 — change it

No getters. No setters. Direct access.
OOP CLASS — private attributes
class Car:
  def __init__(self, name, spd):
    self.__Name  = name
    self.__Speed = spd
self.__Name — WITH double underscore
This means the attribute is PRIVATE.

From outside the class:
c.__Name — BLOCKED (AttributeError)
c.GetName() — must use getter

Needs getters + setters to access.
The ONLY difference — one character
self.Points
PUBLIC — record field
Access: t.Points
vs
self.__Points
PRIVATE — OOP attribute
Access: t.GetPoints()
Summary in one sentence: A record and an OOP class are written the same way in Python. The only difference is that records use self.FieldName (public) and OOP classes use self.__AttributeName (private). When the question says "record" → no underscore, no getters. When it says "OOP class" → double underscore, add getters and setters.

File Processing
Sequential reading is in every paper. Random access is basically never tested. Act accordingly.
🎵 What even IS a sequential file?

Think of a cassette tape. 🎶
You want song 5? Tough — you sit through songs 1, 2, 3 and 4 first. No skipping. Always in order.

A text file works exactly like this. Python starts at line 1 and reads downward. It cannot jump to line 40 without passing through everything above it.

📂
Step 1
Open
Unlock the file so Python can read it
📖
Step 2
Read
Pull text top to bottom, line by line
🔒
Step 3
Close
Lock it back — always do this!
🛠️ Tools You Need — the building blocks
🔓 open("filename.txt")
Opens the file. with open(...) as f: opens AND auto-closes when done.
📖 .read()
Reads the entire file as one giant string. Every character, every newline.
✂️ .split("\n")
Cuts a string into a list at every newline. "a\nb" → ["a","b"]
🧹 .strip()
Removes invisible \n and spaces. "beige\n".strip() → "beige"
📊 .split(",")
Splits CSV lines. "Beech,30".split(",") → ["Beech","30"]
🔢 int() / float()
Cast strings from CSV. int("30") → 30. Must do this for numbers!
🎯 The deal: Every file question uses these tools. Master them and the 4 patterns below will feel like a recipe — not magic.
📦 The 4 Patterns You MUST Know — click to switch between them
Pattern 1 Read everything at once → array ⭐ Appeared: ON 2024 Q1(a) 
# EXAMPLE: Data.txt contains:
# beige
# green
# scarlet
# silver
# bronze

def ReadData():
    Colours = []
    try:
        File = open("Data.txt")
        # Step 1: .read() → "beige\ngreen\nscarlet\nsilver\nbronze"
        # Step 2: .split("\n") → ["beige","green","scarlet","silver","bronze"]
        Colours = File.read().split("\n")
        File.close()
        return Colours
    except:
        print("No file found")

# RESULT: Colours = ["beige", "green", "scarlet", "silver", "bronze"]
What it doesReads the whole file as one string → splits on \n → gives a list where each element is one line.
Mark scheme checks ✓Function ✓ try/except ✓ file opened ✓ .read().split("\n") ✓ file closed ✓ array returned ✓
Pattern 2 Read line by line with a loop Alternative — also valid 
# EXAMPLE: Data.txt contains:
# beige
# green
# scarlet
# silver
# bronze

def ReadLineByLine():
    data = []
    try:
        with open("Data.txt") as f:
            # Loop runs once per line automatically
            for line in f:
                # line = "beige\n" → .strip() → "beige"
                data.append(line.strip())
    except IOError:
        print("File not found")
    return data

# RESULT: data = ["beige", "green", "scarlet", "silver", "bronze"]
#         (same result as Pattern 1 — different method)
Use this when: You don't know how many lines there are, or you want to process each line as you read it.
Key difference from Pattern 1: Pattern 1 reads everything at once then splits. Pattern 2 reads one line at a time and strips. Same result, different method.
Pattern 3 CSV — split by comma → create objects ⭐ Appeared: MJ 2024 Q2(b) 
# EXAMPLE: Trees.txt contains:
# Beech,30,400,200,No
# Oak,25,350,180,Yes
# Maple,20,300,150,No

def ReadData():
    TreeObjects = []
    try:
        File = open("Trees.txt")
        # Step 1: split file into lines
        TreeData = File.read().split("\n")
        File.close()
        for Item in TreeData:
            # Step 2: split each line by comma → ["Beech","30","400","200","No"]
            Parts = Item.split(",")
            # Step 3: cast numbers with int() → create object
            TreeObjects.append(
                Tree(Parts[0], int(Parts[1]), int(Parts[2]), int(Parts[3]), Parts[4])
            )
    except IOError:
        print("invalid file")
    return TreeObjects
The 3-step CSV dance 1. Split file → lines
2. Split each line → parts
3. Cast numbers with int() → create object
Key differenceCreates objects (with all attributes), not just strings. Numbers are cast with int() so you can calculate with them later.
Pattern 4 Read exactly N lines (loop N times) When question gives exact count 
# EXAMPLE: Question says "there are exactly 45 colours in Data.txt"

def ReadNLines():
    data = []
    try:
        f = open("Data.txt")
        # Reads exactly 45 lines (or whatever number question gives)
        for x in range(45):           # ← exact count from question
            data.append(f.readline())   # .readline() = one line at a time
        f.close()
    except:
        print("No file found")
    return data

# RESULT: data = ["beige\n", "green\n", ...]
# ⚠️ WARNING: .readline() keeps the \n — use .strip() if needed!
When to use: The question specifies "45 colours" or "9 trees".
Key difference from Pattern 1 & 2: Stops after N lines — ignores extra. Use .strip() to remove \n if you need clean strings.
⚠️ Important: .readline() keeps the \n at the end. Patterns 1 and 2 remove it automatically.
✍️ Writing to Files — keep it simple
Two modes. Two styles. That's all you need.
Write mode — without with
# "w" = write mode → creates or overwrites
f = open("output.txt", "w")
f.write("Hello\n")
f.write("World\n")
f.close()           # must close manually!
# ⚠ "w" deletes old content!
Reading — with with (auto-close)
# "with" closes the file automatically
with open("output.txt", "r") as f:
    data = f.read().split("\n")
# file is already closed — no f.close() needed
print(data)
Append mode — adds to end without deleting
# "a" = append mode → always seen in exams when adding new records
with open("output.txt", "a") as f:
    f.write("New line added\n")
# "a" keeps ALL old content and adds to the end — safe!
⚠️ Always add \n yourself Unlike print(), f.write() does NOT add a newline at the end. Forget it and all your lines smash together on one line. Muscle memory: always end with \n.

🧊 Binary Files with pickle — the short version
Read all records from a binary file using the sequential pattern.
# ── READ ALL objects (sequential) — the classic pattern ───
import pickle

def ReadAllRecords():
    records = []
    try:
        with open("horses.dat", "rb") as f:
            while True:
                try:
                    record = pickle.load(f)
                    records.append(record)
                except EOFError:        # end of file — stop reading
                    break
    except IOError:
        print("File not found")
    return records
Why the nested try?
Outer except IOError — catches the case where the file doesn't exist at all.
Inner except EOFError — catches when we've read the last record and there's nothing left. That's how you stop the while True loop cleanly.

💡 pickle in one sentence: pickle is for binary objects — it freeze-dries any Python object (with all its attributes) into bytes, saves them, and restores the object perfectly later.

⚠️ Important: If the file has no objects left, pickle.load() raises EOFError immediately. That's how you know you've read everything — catch it and break.
🎯 Random Access — one pattern, that's it
Not in 2024 papers. Know this one example. Move on.
With sequential files, Python reads from top to bottom — it must pass through everything. With random access, Python can jump directly to any position in the file using f.seek(). It's like skipping to chapter 5 of a book instantly.

The key idea: use a hash function to decide where each record goes. Same key → same hash → same position → can find it later without searching. 
# Writing team records to a random-access binary file
import pickle

with open("random_access_teams.dat", "wb") as file:
    for team in teams:
        address = hash(team.TeamName)   # generate file position using hash function
        file.seek(address)              # move file pointer to that position
        pickle.dump(team, file)          # write the team record at that location
Three things happening:
1. hash(key) — converts the team name into a number. Same name → same number, every time.
2. file.seek(address) — teleports the cursor to that exact byte in the file.
3. pickle.dump(team, file) — writes the object at that position. To read it back later: same hash → same seek → pickle.load(file).

Exception Types
The 6 types Cambridge tests — know them cold
ZeroDivisionError
Triggered by: dividing by zero
Mathematically undefined. 10 / 0 → crash. Very common in calculators and grade average programs.
ValueError
Triggered by: wrong value for correct type
Right type, wrong content. int("banana") → crash. The most common one when reading user input.
IOError
Triggered by: file doesn't exist / permission denied
Input/output operation fails. Opening a file that isn't there, writing to a read-only file. Essential for file processing.
EOFError
Triggered by: reading past end of file
Tried to read more data but the file ended. Used in sequential file processing to detect when all records have been read.
ImportError
Triggered by: module not found
import some_module fails because the module isn't installed or doesn't exist. Less common in exams but appears in marking.
KeyboardInterrupt
Triggered by: user presses Ctrl+C
User manually interrupts execution. Useful in long-running programs where you want to handle "please stop" gracefully.

# ── BASIC try / except ─────────────────────────────────────
try:
    number = int(input("Enter a number: "))
    result = 100 / number
    print("Result:", result)
except ZeroDivisionError:
    print("Error: cannot divide by zero!")
except ValueError:
    print("Error: please enter a valid number!")

# User types "0"  → ZeroDivisionError block runs
# User types "abc" → ValueError block runs
# User types "5"  → neither except runs, result = 20.0
# ── EXCEPTION AS e (capture error message) ─────────────────
try:
    result = 10 / 0
except ZeroDivisionError as e:
    print("Caught:", str(e))   # output: Caught: division by zero

# Use this when you want to show the actual error message to the user
# or log it for debugging. Cambridge often asks for this pattern.
# ── FILE PROCESSING with exception handling ─────────────────
try:
    with open("teams.dat", "rb") as f:
        while True:
            try:
                record = pickle.load(f)
                print(record.TeamName)
            except EOFError:
                break                     # end of file → stop reading
except IOError:
    print("Error: file not found or cannot be opened")

# Outer try: catches IOError if file doesn't exist at all
# Inner try: catches EOFError when we've read the last record
# This is the FULL pattern Cambridge expects for file reading
# ── try / except / else / finally ──────────────────────────
try:
    f = open("teams.dat", "rb")
    data = pickle.load(f)
except IOError:
    print("File could not be opened")   # runs ONLY if error
else:
    print("File read successfully!")     # runs ONLY if NO error
    print(data.TeamName)
finally:
    f.close()                            # ALWAYS runs — close the file!
    print("File closed.")
🔎 Why use finally?
finally:
→
This block runs no matter what — whether an exception was raised or not. Perfect for cleanup: closing files, releasing resources. If you open a file in try, always close it in finally.
else:
→
Runs only if the try block completed without any exceptions. Think of it as "the happy path code". Keeps your success logic separate from your error-handling logic.
Why exception handling matters 💡
Cambridge asks you to "describe the importance of exception handling" — know these 6 points.
#PointExplanation
1Prevents unexpected crashesProgram handles the error and keeps running instead of dying abruptly mid-execution.
2Graceful error handlingInstead of a scary Python traceback, the user sees a clear, helpful message.
3Meaningful error messagesYou control what text is shown. "Please enter a valid number" beats "ValueError: invalid literal for int()."
4Improves reliabilityA program that handles edge cases is more robust — it works correctly in more situations.
5Safe continuationProgram can recover from an error and continue safely, rather than leaving data in a corrupt state.
6Essential for file processingDetecting end-of-file (EOFError) and missing files (IOError) is only possible with exception handling.
🏆 The Golden Rule — never skip this Always use try/except around your file operations. Cambridge gives a dedicated mark for exception handling in EVERY file question. It's free marks. Don't lose them.

E2 Write and read records sequentially — with EOFError handling
Intermediate [8 marks] ▼ expand
A program stores StudentRecord objects in a binary file called students.dat. The program needs two procedures: one to write a list of records to the file, one to read them all back.
  • Write procedure WriteStudents(records) that takes a list of StudentRecord objects and writes them sequentially to students.dat. [3]
  • Write procedure ReadStudents() that reads all records and prints each student's name and grade. Use appropriate exception handling for end of file AND handle a missing file. [5]
Cambridge expects: import pickle, correct file modes ("wb" / "rb"), EOFError to stop reading, and IOError for missing file.
Write: open("students.dat","wb"), loop through records, pickle.dump(record, f). Read: outer try/except IOError wraps the open. Inner while True: try: pickle.load / except EOFError: break.
import pickle

# ── WRITE ──────────────────────────────────────────────────
def WriteStudents(records):
    with open("students.dat", "wb") as f:
        for record in records:
            pickle.dump(record, f)

# ── READ ───────────────────────────────────────────────────
def ReadStudents():
    try:
        with open("students.dat", "rb") as f:
            while True:
                try:
                    record = pickle.load(f)
                    print(record.Name, record.Grade)
                except EOFError:
                    break
    except IOError:
        print("Error: could not open students.dat")

E4 Full program — records + file I/O + exception handling
Hard [12 marks] ▼ expand
A program manages a product inventory. Each product has: ProductID (STRING), Name (STRING), Price (REAL), Quantity (INTEGER). Products are stored in a binary file inventory.dat.
  • Declare the record type ProductRecord. [2]
  • Write procedure SaveProduct(product) that appends one product to inventory.dat using binary append mode. Handle the case where the file cannot be opened. [3]
  • Write function LoadAllProducts() that reads and returns a list of all products. Use exception handling for end of file and for a missing file. If the file is missing, return an empty list. [4]
  • Write a main section that loads all products, asks the user for a product name to search, and prints the price and quantity if found, or "Not found" if not. [3]
Append mode in Python: "ab" = append binary. Opens the file and adds to the end without overwriting existing data.
SaveProduct: open with "ab" mode, pickle.dump, wrap in try/except IOError. LoadAllProducts: outer try/except IOError returns []. Inner while True loop with EOFError break.
import pickle

class ProductRecord:
    def __init__(self, product_id, name, price, quantity):
        self.ProductID = product_id
        self.Name      = name
        self.Price     = price
        self.Quantity  = quantity

def SaveProduct(product):
    try:
        with open("inventory.dat", "ab") as f:
            pickle.dump(product, f)
    except IOError:
        print("Error: could not save product to file")

def LoadAllProducts():
    products = []
    try:
        with open("inventory.dat", "rb") as f:
            while True:
                try:
                    product = pickle.load(f)
                    products.append(product)
                except EOFError:
                    break
    except IOError:
        print("No inventory file found — starting fresh")
    return products

products = LoadAllProducts()
search = input("Enter product name: ")
found = False
for p in products:
    if p.Name.lower() == search.lower():
        print(f"Price: {p.Price}  Quantity: {p.Quantity}")
        found = True
        break
if not found:
    print("Not found")
