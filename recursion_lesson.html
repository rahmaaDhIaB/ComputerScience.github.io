01
What is Recursion? — Explained Like You're 5 👶
One idea. Very simple. Don't overthink it.
🪆Imagine Russian nesting dolls.

You open the big doll. Inside is a smaller doll. You open that one. Inside is an even smaller doll. You keep opening… until you reach the tiniest solid doll — the one you can't open anymore. That's the stop.

Then you put them back together, one by one, from smallest to biggest. Done!

That's recursion. A function that calls a smaller version of itself, until it hits the tiny solid doll (the stopping point), then everything comes back up.
📖 Cambridge Definition — learn this word for word Recursion is when a function (or procedure) calls itself as part of its own definition, in order to solve a problem.
🎮Video game analogy:

You're in a dungeon with 5 floors underground.
Every floor you ask: "Am I on floor 0?"
If no → go one floor deeper.
If yes → grab the treasure 💎 and ride the elevator back up through every floor.

Going DOWN = calling yourself recursively.
Hitting floor 0 = base case (stop!).
Coming UP = returning the results back through each level.
02
The 2 Rules — Never Forget These 🧊
Every single recursive function needs BOTH of these. No exceptions.
1
🛑 Base Case — THE STOP SIGN
A condition that returns an answer directly — without calling the function again. This is what stops the recursion.
Without it → the function calls itself forever → CRASH (stack overflow) 💥
Think of it as: "When do I stop asking for help and just answer?"
2
🔁 General Case — THE RECURSIVE CALL
The part that calls the function again — but with a smaller or simpler input. Each call gets one step closer to the base case.
Without it → the function never does anything recursive (it's just a regular function).
Think of it as: "I can't answer yet — let me ask for a smaller version of the same problem."
🏗️ The template — every recursive function looks like this:

def my_function(n):
if [STOP CONDITION]: # ← Rule 1: Base Case 🛑
return [ANSWER] # return directly, no call
else:
return [something] + my_function([SMALLER n]) # ← Rule 2: General Case 🔁 The if is always the base case.
The else always contains the recursive call.
03
Example 1 — Factorial 🧮
The most classic example. Learn this one first.
🧮What is a factorial?

5! = 5 × 4 × 3 × 2 × 1 = 120
4! = 4 × 3 × 2 × 1 = 24
3! = 3 × 2 × 1 = 6
2! = 2 × 1 = 2
1! = 1
0! = 1 ← this will be our base case

Notice something? 4! = 4 × 3! and 3! = 3 × 2!
It's the exact same pattern each time — just with a smaller number!
That's why recursion is perfect here. 🎯
The recursive code 🐍
def factorial(n):
    # ─────────────────────────────────────────────────
    # BASE CASE 🛑  (Rule 1 — the stop sign)
    # When n is 0, we know the answer: 0! = 1
    # We return 1 directly — NO recursive call here!
    # ─────────────────────────────────────────────────
    if n == 0:
        return 1

    # ─────────────────────────────────────────────────
    # GENERAL CASE 🔁  (Rule 2 — the recursive call)
    # n! = n × (n-1)!   e.g. 4! = 4 × 3!
    # We call ourselves with n-1 (SMALLER input)
    # ─────────────────────────────────────────────────
    else:
        return n * factorial(n - 1)

print(factorial(4))   # output: 24

Example 2 — Sum Down ➕
sum(4) = 4+3+2+1 = 10. Same pattern — different problem. Learn BOTH.
➕What does sum_down do?

sum_down(4) = 4 + 3 + 2 + 1 = 10
sum_down(3) = 3 + 2 + 1 = 6
sum_down(2) = 2 + 1 = 3
sum_down(1) = 1
sum_down(0) = 0 ← base case

Notice: sum_down(4) = 4 + sum_down(3)
Same pattern as factorial! Just + instead of ×. 
def sum_down(n):
    # BASE CASE 🛑
    if n == 0:
        return 0       # sum of nothing = 0. Simple. Direct.
    # GENERAL CASE 🔁
    else:
        return n + sum_down(n - 1)   # e.g. sum(4) = 4 + sum(3)

print(sum_down(4))   # output: 10

Example 3 — Count Vowels 🔤
This one is from a real Cambridge paper. Strings work the same way!
🔤Same pattern — but on a STRING instead of a number!

With numbers we went from n → n-1 → n-2 → ... → 0
With strings we go from "house" → "ouse" → "use" → "se" → "e" → ""

Each call chops off the first letter. Base case = empty string. Same idea!
BASE CASE 🛑 When the string is empty ("") there are zero vowels. Return 0 directly.
GENERAL CASE 🔁 Check the FIRST letter. Is it a vowel? Count 1 (or 0). Then call yourself on the REST of the string (Value[1:] = everything except the first letter). 
def RecursiveVowels(Value):
    # BASE CASE 🛑 — empty string, nothing left to check
    if Value == "":
        return 0

    # GENERAL CASE 🔁 — check first letter, recurse on the rest
    if Value[0] in 'aeiou':
        return 1 + RecursiveVowels(Value[1:])   # vowel! count it
    else:
        return 0 + RecursiveVowels(Value[1:])   # not a vowel, skip it

print(RecursiveVowels("house"))    # → 3  (o, u, e)
print(RecursiveVowels("imagine"))  # → 4  (i, a, i, e)

06
The Call Stack — How the Computer Remembers 📚
Every call gets a "box" in memory — stacked like pancakes 🥞
🥞Imagine a stack of pancakes:

Every time factorial calls itself, a new pancake is placed on top.
Each pancake remembers: "I am factorial(n), waiting for factorial(n-1) to finish."

When the base case (factorial(0)) is reached → it returns → the top pancake is eaten → the one below gets its answer → it returns → eaten → and so on until the plate is empty and we have our answer!

Stack Overflow 💥 = no base case → infinite pancakes → the plate collapses!
⭐ Bonus — Advanced Cambridge Knowledge
OutputInOrder — Recursion Without an Explicit Stop 🌳

You know the normal pattern: if n==0: return 1 — the base case is INSIDE the function, clearly written. But Cambridge also uses another style where the stopping condition is a check BEFORE the call. Here's how it works.

def OutputInOrder(RootNode):
    if RootNode.GetLeft() != None:     # 🛡️ only call IF there is a left child
        OutputInOrder(RootNode.GetLeft())

    print(RootNode.GetData())           # print this node

    if RootNode.GetRight() != None:    # 🛡️ only call IF there is a right child
        OutputInOrder(RootNode.GetRight())
🤔 "Where is the base case?! There's no if/else, no return!"
The base case is hidden. Look at the two if conditions. They say:
"Only call myself if the next node EXISTS."

So when you reach a leaf node (a node with NO children), BOTH if conditions are FALSE. The function never calls itself. It just prints and ends. That's the stop.

The base case is: reaching a node with no children (a leaf). It's implicit — it's never written as if ... return, but it's there.
Normal style (factorial) 🛑

Function IS called with the stop value (n=0), then checks and returns. Base case is inside the function.

if n == 0: return 1 ← explicit
Guard style (OutputInOrder) 🛡️

Function checks BEFORE calling itself — if the next value is None, call never happens. Base case is outside, as a guard.

if child != None: recurse ← implicit

For your exams: 90% of questions use the normal style (factorial, vowels, sum). The guard style is just for trees. Don't worry — the core idea is identical. Always ask: "what makes the input smaller?" and "when does it stop?"

R5Write sum of digits — e.g. digits(123) = 1+2+3 = 6
Medium[5 marks]▼ expand
Write a recursive function SumDigits(n) that returns the sum of all digits of a positive integer.

SumDigits(123) = 1+2+3 = 6
SumDigits(45) = 4+5 = 9
SumDigits(7) = 7

Useful tools: n % 10 = last digit. n // 10 = remove last digit.
  • What is the base case? When is n so simple that you return directly? [1]
  • What is the general case? Which digit do you take, and what do you recurse on? [2]
  • Write the complete code for SumDigits(n). [2]
When n has only 1 digit (n < 10), just return n. Otherwise: take the last digit (n%10) and recurse on the rest (n//10).
if n < 10: return n. else: return (n % 10) + SumDigits(n // 10).
# Base case: n < 10 → single digit → return it directly
# General case: last digit = n%10.  Remaining = n//10 (smaller!)
# SumDigits(123) = (123%10) + SumDigits(123//10)
#               = 3 + SumDigits(12)
#                      = 2 + SumDigits(1)
#                             = 1  ← base case
# Back: 1 → 2+1=3 → 3+3=6 ✅

def SumDigits(n):
    if n < 10:                          # BASE CASE 🛑
        return n
    else:                              # GENERAL CASE 🔁
        return (n % 10) + SumDigits(n // 10)

print(SumDigits(123))  # → 6 ✅
print(SumDigits(45))   # → 9 ✅

R8Theory — define recursion and explain its two features
Hard[5 marks]▼ expand
  • Define what is meant by recursion. [1]
  • State the TWO essential features. For each, explain what happens if it is missing. [2]
  • Give ONE benefit and TWO drawbacks of using recursion instead of iteration. [3]
For drawbacks, say WHY — "uses more memory because each call creates a new stack frame."
Definition: "calls itself". Features: base case + general case. Without base case → stack overflow. Benefit: cleaner code. Drawbacks: more memory, stack overflow risk.
# Definition [1]:
# Recursion is when a function/procedure calls itself as part of its
# own definition in order to solve a problem.

# Feature 1 [1]: Base case — a stopping condition that returns a value
# WITHOUT another recursive call. Without it → infinite recursion → stack overflow.

# Feature 2 [1]: General case — calls itself with a SMALLER/SIMPLER input,
# moving towards the base case. Without it → the function never recurses.

# Benefit [1]: Code is shorter and more readable for naturally recursive
# problems (e.g. trees, factorial). The code mirrors the mathematical definition.

# Drawback 1 [1]: Uses more memory — every call creates a new stack frame
# saving local variables. factorial(1000) needs 1001 frames. A loop needs 1.

# Drawback 2 [1]: Risk of stack overflow — if the base case is missing or
# never reached, the call stack fills completely and the program crashes.
R9Recursive binary search — write from scratch
Hard[6 marks]▼ expand
Binary search on a SORTED list: look at the middle element. If it matches → found! If target is smaller → search left half. If larger → search right half.

Write a recursive function BinarySearch(lst, target, low, high)
where low and high are the current search boundaries (start as 0 and len-1).
Return the index if found, or -1 if not found.
  • State the TWO base cases. [2]
  • State the general case — what two recursive calls are possible? [2]
  • Write the complete recursive function. [2]
Base case 1: low > high → not found → return -1. Base case 2: middle element equals target → return mid. General case: target smaller → search left (high = mid-1). Target bigger → search right (low = mid+1).
mid = (low + high) // 2. Three possibilities: found it (return mid), too small (search left), too big (search right).
# Base case 1: low > high → search area is empty → not found
# Base case 2: lst[mid] == target → found it!
# General case: recurse on left OR right half (range halves each call)

def BinarySearch(lst, target, low, high):
    if low > high:                       # BASE CASE 1 🛑 — not found
        return -1

    mid = (low + high) // 2

    if lst[mid] == target:               # BASE CASE 2 🛑 — found!
        return mid
    elif target < lst[mid]:              # GENERAL CASE 🔁 — go left
        return BinarySearch(lst, target, low, mid - 1)
    else:                                # GENERAL CASE 🔁 — go right
        return BinarySearch(lst, target, mid + 1, high)

nums = [2, 5, 9, 13, 21, 34]
print(BinarySearch(nums, 21, 0, len(nums)-1))  # → 4 ✅
print(BinarySearch(nums, 10, 0, len(nums)-1))  # → -1 (not found) ✅
